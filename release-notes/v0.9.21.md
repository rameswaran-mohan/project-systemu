# v0.9.21 — Ship-ready: audited live, hardened, verified

Wave 12. The operator demanded ship quality — so this release was built
by AUDITING the live product (real vault, real recordings, real Docker
containers) and fixing what the audit caught. Every claim below has a run
artifact behind it.

## The recording superpower, proven end-to-end

A real desktop task was recorded through the dashboard (privacy overlay
up), analyzed into accurate plain-English instructions, refined into a
workflow, approved through the unified card, staffed by a newly awakened
specialist, executed — and the deliverable landed in the operator's
output folder with faithful content. The full chain works. Three
ship-blockers found on the way are fixed below.

## Buttons that always respond

Buttons inside live-refreshing lists (Work's **Review & Approve**, the
rail's **Answer**/**Approve**, the Inbox cards) could silently swallow
clicks — the periodic repaint destroyed the button at the moment you
pressed it. Repaints are now change-gated: idle pages stop re-rendering
entirely, and every click lands. Home's **Record** tile now opens the
real recording dialog (it was a dead stub).

## Your first workflow can't dead-end anymore

* On every default install, the "New Shadow Recommended" question after a
  first recording silently auto-skipped — the pipeline stalled at an
  unassigned activity forever. Operator questions now route to the Inbox
  by default under the daemon.
* The wizard's profile save crashed on file-backend vaults — and the SQL
  backends (Docker modes) had the same gap. With v0.9.20's mandatory
  setup gate, both would have LOCKED fresh installs at the welcome
  screen. All three storage backends now share one verified contract.
* Freshly proposed workflows ring the Needs-you badge immediately instead
  of waiting for you to visit Work.
* Read-only commands (`ver`, `dir`, `git status`…) now execute instead of
  being auto-denied by a name-based safety gate; denials that do happen
  teach the agent to adapt instead of looping.
* An objective claimed on a failed attempt is no longer lost — the agent
  is nudged to re-claim it once the work actually succeeds, so finished
  runs stop being cancelled and retried.

## Works out of the box

* All six pip packages the 41-tool starter pack declares now ship with
  `pip install systemu` — no dependency-approval interruptions on first
  use. A permanent conformance gate executes the pure-local pack tools
  through the real runner with outcome assertions.
* Catalog model ids like `google/gemini-3-flash-preview` now work with
  only an OpenRouter key (key-aware provider fallback; explicit provider
  overrides still win).
* Docker: `docker-local` verified end-to-end in containers (postgres
  healthy, dashboard serving, worker up); `docker-enterprise` compose
  config validated.

## Faster everywhere

Every page build carried a ~1-second process scan from the health
banner. It's now cached: Home/Work/Inbox/Welcome build in **50–75 ms**
(was 1.2–2.0 s). Plus: the sidebar footer shows the real port, timezone
detection yields IANA names, and the watchdog window is tunable for slow
models (`SYSTEMU_STUCK_THRESHOLD_S`).

## Read this one page

**OPERATOR-SOP.md** ships with the install: the one clear path
(install → ask or teach → approve → watch → results), what each approval
card means, where everything lives, and a troubleshooting table. Linked
from the README and Settings → Help.

## Verification

Full suite at release: 3,200+ tests, 0 regressions; the starter-pack and
golden-task gates run keyless in CI. Live verification: first-run wizard
+ tour, quick task to deliverable (~3 s), the full recording pipeline,
and containerized boot — all exercised on a real machine against real
vault copies, with findings (including the honest leftovers: alembic on
fresh postgres falls back to create_all; destructive confirms auto-deny
safely rather than asking the Inbox) recorded in the audit report.
