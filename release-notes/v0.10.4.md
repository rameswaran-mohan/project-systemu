# v0.10.4 — Receipts you can see: verified/claimed badges

The G-DEMO milestone (v0.10.0) proved the daemon can act through a connected tool and then
*independently* read the effect back to produce a machine-verified receipt. But that receipt only lived
in memory for the duration of the run — nothing showed it. This release makes it **visible and durable**.

## What ships

- **Receipts are now durable.** When the daemon acts through a tool and reads the effect back, the
  resulting receipt (verified vs claimed, and how it was verified) is written to a durable per-run store
  that **survives the run's completion** — so you can look at a finished run and see whether its external
  effects were independently checked.
- **A "Receipts" panel on the run detail page.** Each external effect shows a **Verified** badge (the
  effect was independently read back and machine-checked — *receipts, not self-report*) or a **Claimed**
  badge (the tool reported it, but it wasn't independently verified). A run with no external effect shows
  no panel — never a fabricated badge. The badge is separate chrome from status and cost: it says the
  effect was verified, not that the run succeeded.

## Security posture (why this is safe)

The durable receipt is **display-only**. It is written *alongside* — never instead of — the live in-run
evidence the credit gate reads, and it is read by **nothing except the UI**. So a tampered or forged
receipt file can never cause an effect to be credited: crediting still depends solely on the live,
in-run verification. (For the same reason it is deliberately not written into the action-audit store,
which other logic reads.) The run's own execution id is path-sanitized before it becomes a filename.

## Scope

- Purely additive: the credit path is byte-identical, proven by test. Full suite green.
- The badge renders the R-A14a MCP-actuation receipt (which is produced today, with the verification net
  off). Money-moving effects continue to show **Claimed** until their dedicated freshness probe lands —
  a money-move is never shown as verified without independent, provably-fresh proof.
