# v0.10.20 — Big event details can't drop the dashboard connection (R-UX2)

A dashboard robustness fix, found by running the app: a single event carrying a very large tool
result, parameter dump, or reasoning trace could bloat one live-feed update past the WebSocket
message limit — showing "Connection lost — the message is too large for WebSocket transmission."

## The fix

Each rendered detail block in the Live / event feeds (tool result, tool params, reasoning, outcome
summary) is now capped at ~8 KB with a clear truncation note — the full record stays available on the
run's own page. No single event can inflate a feed update enough to break the socket.

The live event list was already bounded (last 50 events) and the WebSocket max size already raised;
this closes the remaining unbounded path — a single oversized detail element. Pure clip logic
unit-tested; existing feed tests unchanged. Full suite green.
