# v0.9.36 — harness reliability, dashboard polish, and open-source packaging

A reliability + polish release. Two harness-runtime fixes from the
capability-gap test matrix, a dashboard fix for the live activity indicator,
and a pass of documentation/packaging hygiene for the public release.

## Fixed — harness terminal finalize runs exactly once per run

The terminal harness finalize (request-outcome reconciliation + lease revoke +
MCP tool teardown) previously fired only on the COMPLETE / FAIL / suspend /
goal-verified exits. The partial, max-iterations, exception, and
suspend→approve→resume-completion exits skipped it, which caused two problems in
long-lived processes:

- **Cross-run MCP tool leak** — a server's namespaced tools, registered for one
  run, could linger in the process-global tool registry and reappear in a later
  run's catalog. The terminal teardown now runs once per run from the
  execution's `finally` block (idempotent), and additionally unregisters every
  MCP server the run registered, independent of lease bookkeeping — so a
  resumed run (whose lease lived in the pre-suspend governor) no longer leaks.
- **Missing request-outcome instrumentation** — the granted/used/unused/wasted
  reconciliation is now written on every terminal exit.

Suspends (which resume) are explicitly excluded from the teardown so a parked
run keeps the capabilities it will need on resume.

## Fixed — `attempts_before` and `confidence` on harness requests

The runtime already captured these two fields, but the executor prompt never
documented them, so they were always null. The `REQUEST_HARNESS` schema +
example now include them, restoring the request-quality signal.

## Fixed — Live activity dots not visible

The small "working in the background" dots next to the dashboard's **Live**
header (and in the live-events pane) had no explicit color and fell back to a
muted default that was hard to see on the dark rail. They now render in the
accent color.

## Changed — documentation + packaging hygiene

Pre-public-release polish: corrected a couple of documented model identifiers to
match the shipped defaults, added an explicit `license = "MIT"` to the package
metadata, routed two analyzer functions' default model through the standard
tier/preset resolution (instead of a stale hardcoded id), and generalized
example output in older release notes. No behavior change for configured runs.
