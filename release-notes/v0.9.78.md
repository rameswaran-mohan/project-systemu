# v0.9.78 — R-A14a (core): verifiable MCP actuation

The first slice of tier-2 MCP-only actuation — the foundation of G-DEMO v0 ("open a GitHub issue from
this bug note"). When the agent acts through a connected MCP tool that creates a resource, the system
can now **independently read that resource back to produce a receipt** — turning "the tool said it did
it" into "the effect was machine-verified." This works with the verification net **off** (the default),
so the receipt doesn't wait on arming the net.

## What ships

- **An actuation contract + verifiable MCP mutations.** A new modality layer wraps the existing
  MCP-call path so that a resource-creating MCP mutation is independently verified (an HTTPS read-back
  of the created resource), yielding a **verified** receipt when confirmed and a **claimed** marker
  when it can't be — evidence-first, not self-report. The effectful tool is never re-invoked; the
  read-back is a separate GET.
- **Money-moving MCP actuations are held to the strict bar.** A money-moving MCP effect is credited
  only via an independent, host-pinned, provably-fresh read-back — never a self-reported or advisory
  signal. (In this slice a money-move MCP effect therefore stays uncredited until its dedicated
  anti-replay probe lands — a deliberately safe posture: it can never be wrongly credited.)
- **No behavior change with the net off, for everything else.** The independent reader is present but
  dormant for non-MCP effects when the net is off; non-MCP runs are byte-identical, and a
  non-money MCP mutation credits exactly as before (now with a best-effort receipt attached).

## Compatibility & scope

- **Read/verify only; the verification net stays off by default.** MCP actuation verification is
  decoupled from the net's arming, so it runs now without changing any non-MCP behavior. The full
  suite (5524 tests) is green. A six-lens adversarial review confirmed the money-move invariant holds
  and fixed one default-config regression (non-money MCP mutations were briefly un-creditable) before
  release.
