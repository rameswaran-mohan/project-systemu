# v0.9.69 — R-A11b-2 hardening: tool-reuse resolves strictly by id

A small defense-in-depth follow-up to v0.9.68's discovery-before-forge auto-reuse, surfaced by a
re-verification pass.

## What changed

When the runtime reuses an existing tool (instead of forging a duplicate), it now resolves that
tool **strictly by the id it re-verified** at grant time. Previously the resolver could fall back
to a name lookup, and — in an extremely narrow window where the verified tool vanished between the
re-verify and the hand-off while a not-yet-reviewed tool of the same name happened to exist — the
name lookup could pick that unreviewed twin. A reuse now treats a vanished id as genuinely stale
(the agent re-requests) rather than substituting a same-named tool. The normal forge path is
unchanged.

No behavior change for any ordinary run; the full suite (5158 tests) is green.
