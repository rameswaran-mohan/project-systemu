# v0.9.48 — forged file-tools: validate, self-heal, or fail fast

When a task forged a tool that operates on a real document — *"hash and
password-protect `report.docx`"* — the tool could deploy broken, run, fail, and
leave the task **stuck** with no feedback. The dry-run couldn't validate file
tools, the enable gate was inconsistent, nothing finalized the parked task, and
the "task blocked" card sometimes crashed. This release closes all of that and
adds a **bounded, governed self-heal** so a fixable forge bug repairs itself once
instead of dead-ending.

## Fixed

- **The dry-run now validates file/format tools.** It synthesizes a format-valid
  fixture (`.docx` / `.xlsx` / `.pdf` / `.png` / `.zip`, text otherwise) for
  path-like parameters and recognizes `source_path` / `*_path` / `*_file` keys —
  so a *correct* document tool actually passes dry-run instead of being doomed by
  a dummy-bytes input. Formats the harness genuinely can't synthesize fall to an
  **operator-verify skip**, never a hard failure.
- **A tool that can't deploy fails the task fast — with feedback.** A parked task
  whose required tool can never pass dry-run is now finalized *failed* with the
  dry-run error surfaced (two independent safety nets: the Inbox handler and a
  daemon-side reaper), instead of hanging forever after "Enable & run".
- **One enable gate, no laundering.** Every enable path now goes through a single
  gate that refuses a `failed` dry-run; the recalibration "Enable & Resume" path
  no longer quietly rewrites `failed → skipped`, and a tool that newly fails its
  dry-run is auto-disabled so it can't stay callable.
- **Non-conforming forged code is rejected before it hits disk.** A `run()` whose
  required parameters can't be filled from the declared schema is caught at forge
  time, not at first call.
- **Governed self-heal on a dry-run failure.** When a forged tool fails its
  dry-run with a *code* bug, the Governor authorizes exactly **one** re-forge that
  feeds the dry-run error back into the code prompt as an authoritative
  correction. It runs in-line on the reconcile tick (no loop, no added wait); a
  missing-dependency or permission failure still routes to you instead.
- **The "task blocked" card no longer crashes.** The decision-card renderer is now
  serialization-safe end-to-end (no more `Type is not JSON serializable`).
- **No more false "missing argument" dry-run failures.** When the test-parameter
  generator returns an empty or partial result, the missing parameters are now
  backfilled from the schema defaults, so the tool is exercised with a complete
  call.

## Under the hood

Root-cause-first across the forge → dry-run → enable → finalize spine, plus a
shared schema-normalization helper that unwraps a wrapped `{type, properties,
required}` parameter schema. Every fix landed test-first (≈90 new tests).

Note: a forged tool can only be as correct as the model that writes it — a tool
whose bug is a genuine library-API misuse the configured model doesn't know may
still fail its dry-run, in which case it now fails fast with the error rather than
deploying broken.

No eval/paper changes.
