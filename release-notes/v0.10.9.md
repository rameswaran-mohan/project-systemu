# v0.10.9 — Don't drop a distinct file choice from the confirm bundle

A small correctness fix. When the system needs the operator to confirm a set of choices in one click
(e.g. which file each of several steps should use), two steps that pointed the **same field** at
**different values** were being collapsed into a single item — so the second choice silently vanished from
the bundle. The de-duplication now keeps distinct bindings distinct, while still collapsing genuinely
identical asks. Full suite green.
