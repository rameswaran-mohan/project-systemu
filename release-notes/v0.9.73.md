# v0.9.73 — R-A13b-2iii: the shadow park-surface report + readiness gate

The measurement side of turning the external-verification net on now has an **observation surface**
and an **executable readiness check**. The shadow meter (v0.9.71) records how the net would behave;
this release lets you *read* that measurement and get an at-a-glance verdict on whether the net is
ready to arm. Read-only; the net stays off.

## What ships

- **A shadow park-surface report.** A new diagnostic — `systemu debug s4-shadow-meter` — shows, per
  effect type, how the external-verification net *would* have behaved in shadow mode: how often it
  would have confirmed an effect, how often it would have parked one for operator review, and the
  resulting park rate.
- **An executable readiness verdict.** The report ends with a plain READY / NOT-READY verdict for
  arming the net, on a criterion that is deliberately *not* "zero parks" (some effects are genuinely
  unverifiable and should always park). Instead it checks that there is enough measured data, that no
  benign effect is being needlessly gated (which would mean effect classification is unfinished), and
  that every gated effect type actually has a working confirmation channel. Today, against a
  not-yet-classified inventory, it correctly reports NOT-READY with the specific reasons.

## Compatibility & scope

- **Read-only, no behavior change:** the report reads the existing shadow-meter counters and writes
  nothing; it changes no run outcome in any mode. The full suite (5277 tests) is green.
- Populating the effect classification the readiness verdict checks for is the next step.
