# v0.9.14 — Dashboard bugfixes (live events, detail arrows, stuck tasks)

A patch release fixing four dashboard regressions reported against v0.9.13.
The important one is a silent tool-execution failure on Windows.

## Fixes

- **Tasks no longer stall after activity extraction (Windows).** The dashboard
  is served by uvicorn, which runs on a `SelectorEventLoop` on Windows — where
  `asyncio.create_subprocess_exec` is unsupported and raises
  `NotImplementedError`. v0.9.13's "forged tools run out-of-process by default"
  change routed those tools through the subprocess path, so every forged-tool
  call failed **silently** and the run stalled. The tool subprocess now runs via
  a worker thread + `subprocess.run`, which is event-loop- and
  platform-agnostic. (Verified: a real subprocess tool call now succeeds on both
  the Proactor and Selector loops.)

- **Live events stream without a page refresh again.** The live-events pane only
  started its refresh timer when the websocket was already connected — but at
  page-build time it never is, so the timer was never scheduled and the feed
  showed only its initial replay until you reloaded. The timer is now scheduled
  directly (and still cancels cleanly on disconnect).

- **The per-event expand arrow is back.** The Home rebuild in v0.9.12 dropped the
  live activity panes — the arrow that exposes an event's reasoning, tool
  parameters, tool result, and (lazily) the raw LLM response. Restored on Home,
  and the same expand-arrow now also appears on the `/chat` Live Events feed.

- **Active Tasks**: confirmed working — the badge and menu populate from running
  jobs. (It read as broken only because a stalled task — the bug above — is not a
  running job, so the menu was correctly empty.)

## Verification

7-test regression suite added, including the load-bearing case (a subprocess
tool call must succeed on a `SelectorEventLoop`). Full suite: 2,853 passed,
zero new failures — and two long-stale baseline tests now pass as a side effect
of restoring the live panes.
