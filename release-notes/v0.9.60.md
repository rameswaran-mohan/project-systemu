# v0.9.60 — R-A9 + R-A10: the open-world planner, situational inventory & requirement-binder

**The agent now surveys its situation, reasons a plan over it, and computes "what's missing" as a
schema-diff instead of a guess.** This bundles two milestones of the Open-World Go-Getter build —
**R-A9 (A-Inventory, spec §5.1)** and **R-A10 (open-world planner §5.2 + requirement-binder §5.3)** —
onto the G1 persistence substrate (v0.9.59). Together they turn a scroll-follows-the-script executor
into a run-time planner that inserts prerequisite objectives, binds required parameters from what it
can see, and recovers from auth/semantic failures by asking for exactly what it lacks.

Every replanning-free run remains **byte-identical** to before (the AC6 floor): the new stages are
strict no-ops unless the situation actually calls for a plan change.

## R-A9 — Situational inventory (§5.1)

- **A per-run `SituationReport`.** Before the loop, the runtime surveys five sources — credentials
  (names only), the operator profile, connected MCP services, deployed+enabled capabilities, and the
  granted-roots filesystem (a bounded, junction-safe BFS) — into one report. Each entry carries an
  **immutable taint axis** `origin_class ∈ {operator, systemu_authored, content_derived}`.
- **Fail-safe & non-blocking (AC7).** The survey runs off-loop on a dedicated executor with
  per-source timeouts and an overall bound; any slow/raising builder degrades to a cached-or-empty
  slice and never blocks or fails the run.
- **A hardened `fence()` (AC6).** Untrusted inventory text rendered into any prompt is wrapped in a
  per-call-nonce delimiter, with embedded close-delimiters neutralized and non-str payloads
  fail-closed — a report can describe the environment but can never inject instructions.
- **Freshness-stamped + resume-cached.** Per-slice stamps drive cache invalidation; a resume reuses
  unchanged slices (AC3). Persisted via `ExecutionSnapshot` schema **v3** (`_migrate_2_to_3`).

## R-A10 — Planner, binder, and requirement-fold

- **Run-time open-world planner (§5.2).** A pre-loop, Tier-1 reasoning stage reads the *fenced*
  inventory and may insert **precede-objectives** (e.g. "obtain a credential", "resolve a
  prerequisite") ahead of the goal's named objective — not limited to the goal's own service. Inserts
  allocate a monotonic id, stamp `origin="planner"`, and wire `depends_on` so the captured outcome
  waits. Fresh-runs-only, fully fail-safe (any error → the static tree), and a *no-insert* decision
  leaves the schedule untouched by identity. `origin="planner"` grants **no** elevated trust — every
  inserted objective is governed per-tool-call exactly like any other.
- **Requirement-binder (§5.3): "what's missing" is a schema-diff, not a guess.** The binder reuses the
  deterministic `fixture_synth` schema walker (the 26 shape tests stay byte-identical) to diff the
  chosen capability's schema and bind each required leaf from five sources (schema default, run
  context, the situation report, the operator profile, prior outputs). Taint travels verbatim
  (IMPL-5): a `content_derived` value is **never** silently bound — it always becomes a one-click
  confirm — and an untrusted file/inventory origin is clamped to `content_derived` rather than
  trusted. Unbound required leaves become a typed `RequirementReport.ask_bundle`.
- **Runtime-error-as-requirement fold (AC4).** A tool failure that is really a *missing requirement* —
  a 401/403 (credential) or a 422/404 (decision) — folds into a requirement + a backchain
  precede-objective and **suspends for the operator** instead of counting toward the stuck bound and
  failing the run. On resume the supplied credential re-dispatches the tool and credits the precede.
  The credential gate is enforced at **every** objective-credit site: an objective gated on a
  still-missing requirement can never be marked complete — no unauthenticated "success".
- **Durable, mergeable objective graph.** The authoritative post-fold graph is written back
  conditionally (only when it diverged from the static tree — the AC6 guard) and rehydrated on
  resume. A resume that carries **both** a persisted mutated graph **and** an operator
  param-substitution now **merges** them: the graph stays authoritative for structure/inserts while
  the operator's substitution is replayed over every value leaf. Snapshot schema **v4**
  (`_migrate_3_to_4`).
- **Single-card ask rail + MODEL-MATRIX.** A `RequirementReport` requirement renders through the
  existing elicitation rail (one card, secrets auto-routed URL-mode, bound value pre-filled). A pinned
  `docs/MODEL-MATRIX.md` (DEC-12) + `Config` tier knobs (`planner_tier`/`binder_tier`/`parser_tier`)
  route each LLM stage to its intended model class, with a `locality` column (DEC-20a) for the
  privacy story.

## Compatibility & scope

- **AC6 byte-identical floor.** A goal that triggers no replanning produces a byte-identical schedule
  and snapshot to v0.9.59 — verified by a full-run differential. All new stages early-return on the
  no-change path.
- **Additive, migrated durable state.** `Objective.requirements` (default `[]`) and the new snapshot
  fields validate against older on-disk snapshots; the `SnapshotMigrator` chains v1→v2→v3→v4,
  migrating older and honestly refusing newer.
- **Deferred to R-A12:** the batched multi-requirement scope card + re-plan-on-resume, and the
  mid-loop binder invocation with a live per-objective capability (today the producer seam is wired
  as an AC6-safe no-op). **Deferred to R-A13.5:** the `fixtures/field/` replay corpus (its format is
  defined by that harness; the R-A9/R-A10 scenario schemas are a tracked backfill debt).

Full suite green (4635 passed). Every step was built test-first and reviewed across spec, code-quality,
and adversarial lenses; the adversarial lane repeatedly caught real defects — an unauthenticated-
completion class in the fold, a fence-escape, and a param-sub merge crash — that green suites missed.
