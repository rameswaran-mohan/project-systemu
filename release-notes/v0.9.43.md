# v0.9.43 — operator-input loop, one-click reject (×), compact "Needs you" (+ forge re-dispatch fix)

Three runtime/UX fixes for the chat → forge → answer loop the dashboard demos.

## Fixed — forging + enabling a tool no longer leaves the task stuck

The readiness gate promised *"the heal sweep re-runs the parked task once tools
are ready,"* but the heal sweep (`heal_activities_for_tool`) was only ever called
from the Tools page. Forging a tool and approving **Enable & run** from the
**Decisions Inbox** (the demo path) enabled the tool and then stopped — the parked
task never re-dispatched.

The `tools_blocked` resolver now fires the heal sweep for each enabled tool (in a
background thread, since it makes LLM calls), so the parked activity resumes and
runs to completion. Idempotent: the heal only re-dispatches an activity once all
its required tools are ready.

## Added — the quick lane can ask you a question and *resume*

A quick chat task that needed a value (*"find the SHA-256 of any word I provide"*)
used to emit `needs_input` and dead-end — the question showed, but there was no way
to answer it.

Now, when a live chat operator is present, an `ASK_USER` **pauses** the run on an
answerable question in **"Needs you"** and waits for your typed answer (the same
block-and-wait the command-approval gate already uses — the run's loop state stays
intact on its worker thread, so no work is lost). Your answer is fed back into the
run as a `tool_result` and the run **continues to a real result**. The model's
prompt contract is updated (asking pauses, not ends, the run; the answer returns as
a `tool_result` — never re-ask), and a 3-question cap prevents re-ask loops.
Non-chat / CLI / headless callers keep the historical `needs_input` terminal.

## Added — one-click reject / decline (×) on every "Needs you" card + a compact rail

Every card in the "Needs you" rail now carries a small **×** that performs the
card's *safe negative* — Dismiss / Skip / Deny for gates, **Decline** for
questions — symmetric to the existing one-click Approve. It records the negative
resolution and unblocks any parked run.

- Command / MCP gates (previously rail-render-only) now get a **one-click Deny**
  via the × — a one-click Deny is safe; only one-click *Approve* still routes
  through the full `/insights` Inbox.
- The rail is tightened — smaller title and action buttons, less padding — so more
  gates stay visible before the panel scrolls.
