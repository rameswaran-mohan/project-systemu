# v0.9.62 — R-SEC1: dashboard authentication (fail-closed non-loopback gate)

**The dashboard can no longer be served unauthenticated to a network.** Until now the
NiceGUI dashboard had **no authentication of any kind**, and every Docker profile published
it on `0.0.0.0:8765` — an unauthenticated control plane reachable by anyone who could reach
the port, behind which every S1b action gate is bypassable. This release adds session
authentication with a **fail-closed exposure rule**, so a network-exposed dashboard is
either passphrase-protected or it does not start.

This is a hotfix-class security release and a hard precondition of any partner install or
non-loopback deployment. **The local, loopback-only experience is unchanged** — no
passphrase, no friction, byte-identical to before.

## The rule (fail-closed)

- **Loopback bind (`127.0.0.1`, `::1`, `localhost`) — auth optional.** The dashboard starts
  as before; if no passphrase is set it logs a one-line "UNAUTHENTICATED (loopback only)"
  warning. This is the default local-dev path and it is untouched.
- **Non-loopback bind (`0.0.0.0`, a LAN IP, …) — auth required.** If no passphrase is
  configured, the dashboard **refuses to start** with an honest, actionable message rather
  than silently exposing an open UI. Configure a passphrase and it starts with the session
  guard active.

## What ships

- **Passphrase auth, stdlib-only.** `hashlib.scrypt` (n=2¹⁴, r=8, p=1, 32-byte random salt);
  constant-time `verify` that never raises (any parse error is a fail-closed `False`). No
  new dependency. The hash (and the session secret) are stored via the S5 DPAPI at-rest
  envelope where available, `0600`-guarded otherwise.
- **Session guard that walks the live route table.** A middleware requires an authenticated
  session for every route except a minimal, **segment-boundary** allowlist (`/login`,
  favicon, and the NiceGUI/`/assets`/`/static` infra prefixes). A test enumerates the real
  registered routes and asserts each non-allowlisted one is guarded, so a newly-added page
  can never ship unguarded. HTML navigation → 302 `/login`; XHR/API → 401.
- **Brute-force controls.** Per-IP **and** global failure counters (persisted); 5 failures →
  15-minute lockout, checked before the passphrase so a correct guess can't slip in during
  the window; the counter resets cleanly after each window; a distributed spray across many
  IPs trips the global lockout.
- **Config paths.** `systemu doctor --set-passphrase` (prints the
  `SYSTEMU_DASHBOARD_PASSPHRASE_HASH=…` line for headless/Docker), or set that env var
  directly. **TLS passthrough:** `SYSTEMU_TLS_CERT`/`SYSTEMU_TLS_KEY` are forwarded to the
  server; a non-loopback bind without TLS logs a plaintext-exposure warning.
- **Honest capability state.** `dashboard_auth: session | none(loopback-only)` and
  `tls: on|off|n/a(loopback)` are exposed for the health/privacy surface.

## ⚠️ Breaking change (intended) — passphrase-less Docker now refuses to start

Every bundled Docker profile binds `0.0.0.0`. With this release a container started **without**
a passphrase will **exit non-zero** on boot instead of serving an open dashboard — this is the
fix, not a regression. To upgrade a Docker deployment:

```
# generate a hash (locally or in the container)
systemu doctor --set-passphrase        # prints SYSTEMU_DASHBOARD_PASSPHRASE_HASH=scrypt$...
# then provide it to the container, e.g.
export SYSTEMU_DASHBOARD_PASSPHRASE_HASH='scrypt$14$8$1$...'
docker compose --profile local up
```

The compose profiles now reference `${SYSTEMU_DASHBOARD_PASSPHRASE_HASH:-}` so the value flows
from your environment. A purely local (loopback) install needs nothing.

## Compatibility & scope

- **Regression floor (verified).** With no passphrase configured on the loopback default, the
  guard is a strict pass-through no-op and the entire existing UI test suite passes **with zero
  edits** — the local experience is byte-unchanged.
- Every step was built test-first and reviewed across spec, code-quality, and **two adversarial
  security passes** (skeptic-verified). Those passes caught real defects the green suites
  missed — a plaintext session secret (session forgery), an env-only-config path that armed the
  guard but made login impossible, an unbounded-prefix allowlist bypass, a lockout counter that
  never reset, and a corrupt-secret-file fail-open — each fixed fail-closed before release.
- **Deferred:** RBAC/SSO (COMPLIANCE §CMP-0.b, Track-F), `doctor --make-cert` self-signed
  generation, and the dedicated `/health`+privacy-page chips (land with R-UX1).
