# v0.9.55 — OnTheTable: read-only inventory board (T1)

**OnTheTable — the operator-facing half of the Situational Inventory (spec
`UNIFIED-v2` §5.10). An existing install now sees "what it has" — MCP servers, the
tool catalog, and stored-credential names — projected into a durable, deduped,
healable set of TableItems and rendered on a read-only `/table` board, with zero
operator input.** Additive: no gate/planner consumes the projection yet (curation +
the consult are T2/T3), so no runtime behavior changes. This release is the backend
(T1a) plus the read-only page + daemon job (T1b).

## What ships

1. **`runtime/table_store.py` — model + side store.**
   - `TableItem` (pydantic): `kind / name / status / provenance / origin_class / ref
     / parent_id / usage / pinned`, with `origin_class` **immutable** through status
     transitions (the §5.10.b taint rule) and `ref` holding operational
     ids/names/paths only — **never a secret value**.
   - Side store at `<vault>/table/items.json` (+ `tombstones.json`), copying the
     `mcp/connections.py` defensive-read pattern (a broken/absent file ⇒ empty state,
     a malformed entry is skipped — the page shell can't die on it) but with **atomic
     writes** (tempfile + `os.replace`).
   - `ref_key()` — stable per-kind identity (server URL / tool id / normalized path /
     credential name), so a **rename can't fork an item** and a re-added object
     **heals** its existing item.

2. **`runtime/table_reconciler.py` — deterministic projection.** `project(vault)`
   maps three live stores into TableItems: MCP servers + their enabled-tool count
   (`mcp/connections`), the vault tool catalog (`vault.list_tools()` — status
   ready/broken/declared, `origin_class` by `forged_by_systemu`, `effect_tags` carried
   in `usage` once G0 lands), and the credential **name** registry. Idempotent (stable
   id = hash of `ref_key`), **dedups**, **respects tombstones** (operator-removed refs
   never re-added), and **preserves operator curation** (`pinned`, `usage`, `created_at`)
   across re-projection. `reconcile_once()` projects + persists; never raises.

3. **Credential NAME registry** (`runtime/credentials/store.py`) — a names-only
   sidecar (`.credential_names.json`) maintained on `set`/`delete`, because the keyring
   backend (Windows Credential Manager) **cannot enumerate** its own entries.
   `list_names()` is the projection surface. Never holds a value.

4. **Read-only `/table` page** (`interface/pages/table.py`) — a zoned board (Services
   & Accounts · Tools & Capabilities · Files & Data · Keys · Preferences · Devices)
   rendering the projection: per-item status dot, kind/provenance chips, a count
   summary, and an empty state. Spine-less route (like `/inbox`/`/welcome`), reachable
   via an "On the table" link in the header. All values crossing into nicegui are
   plain strings (the v0.9.45 serialization-crash rule). The zoning/summary logic is
   pure + unit-tested; the render is operator-verifiable.

5. **Projection reconciler daemon job** — `table_reconciler.reconcile_once` registered
   as a 60s interval job (matching the `tool_reconciler` pattern), keeping
   `<vault>/table/items.json` warm. Never raises; the page also projects on demand, so
   the job is warmth, not correctness.

## Design guarantees

- **Never subtracts.** Projection only *maps* the live stores; when the Situational
  Inventory consumes it (a later release) it can annotate/re-rank but never hides a
  live store object — the Callout-3 "look at everything" floor.
- **Read-only + additive.** The page displays; nothing curates, gates, or plans from
  it yet. On the plain v0.9.52 base, tool `effect_tags` are empty until G0 (v0.9.53)
  merges.

## Tests

- `tests/test_onthetable.py` (16) — **backend:** model defaults; store round-trip
  through the JSON boundary; defensive load on a broken file; malformed-entry skip;
  `ref_key` identity (slash-normalized, id-over-name); tombstones; the credential name
  registry (names-only, set/delete); projection from live stores (kinds, status
  mapping, effect_tags in usage); idempotent stable ids; tombstone-respect; pinned
  survives re-projection; `reconcile_once` persists. **page:** module imports;
  `zone_of` mapping; `group_into_zones` drops-empty/orders; `summarize` counts.
- Full suite green (4249 passed, 22 skipped, 1 xfailed — baseline 4233 + the 16 new tests).
