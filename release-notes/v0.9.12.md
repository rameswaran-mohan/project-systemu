# v0.9.12 — UI Revamp Phase 6: Cleanup & Naming (the revamp is complete)

**Release date:** 2026-06-10
**Headline:** The final phase of the six-phase UI/UX revamp — cleanup, consistent naming, and a real **Home** spine. Dead surfaces are gone, the legacy routes are true server redirects, the duplicate approve/decision surfaces are unified, and `/` is now an at-a-glance command-center Home instead of the old console. With this, the revamp (design system → shared command layer → Decisions Inbox → IA shell → spine consolidations → cleanup) is **done**.

> Phase 6 of the UI/UX revamp; builds on v0.9.11 (spine consolidations). UX + internal-architecture release — agent behavior is unchanged. Regression-clean (**0 new test failures** vs the pre-existing baseline; **0 new** UI-style-lint violations), live-smoked, and validated with a real end-to-end agentic task.

## What landed

### A real Home spine
`/` is no longer the heavy Console page retitled — it's a lean **at-a-glance dashboard of links**: a "What needs you" summary (pending-gate count + top titles → the Inbox), "What's running" (the workflow pipeline → Work), spine quick-tiles, recent activity (each → its workflow), and the pending-approvals queue. The old in-page **re-renders** of other pages' content (the duplicate decision cards, learning-curves / memory / skills panels) are gone — Home links out to Insights/Shadows/Inbox instead of duplicating them.

### Naming & dead-code cleanup
- **`/shadows` is now the canonical route** (`/army` → 307 redirect); the `shadow_army` storage contract is unchanged.
- Page titles align with their nav labels (Home, Build).
- Dead `overview.py` shim removed; **`/workshop` fully removed** — its last surface (the interactive scroll rebuild) moved into an in-place dialog on the Scrolls/Work surface, with the underlying `rebuild_scroll` pipeline unchanged.

### One surface, not many
- The **legacy blind scroll-approve** (the notifications "Pending Actions" path) now routes through the **unified inspect-before-approve card** — so every scroll approval, everywhere, shows risk · inspect · what-Approve-does before you act.
- The 5 legacy routes (`/army`, `/memory`, `/flywheel`, `/notifications`, `/systemu-chat`) are now **true HTTP 307 server redirects** (not client-side hops) to their canonical homes.

### Robustness & polish
- Silent-swallowed errors that hid a real state mutation (a dropped notification, an unrecorded audit event) now **log** — while the legitimately-defensive render/disconnect guards are left intact (so the dashboard still degrades gracefully on transient hiccups).
- **＋New → "Submit task"** now surfaces a live **"View in Work"** link to the new workflow when the task completes.

## Tests & verification (honest)
- New suites for the Home summary models + the Phase-6 substeps (titles, redirects, the notifications-gate routing, the scroll-rebuild dialog).
- **Full regression: 0 new failures** vs the pre-existing baseline; **UI lint: 0 new** (Home's rebuild actually *removed* pre-existing inline-style debt).
- **Live smoke:** `/shadows` is canonical (200), `/army` + the 4 legacy routes return 307 redirects, `/workshop` is 404, and Home renders the at-a-glance layout.
- **Live agentic validation:** a real task ("create a file with exact content") ran end-to-end on the live engine and produced the **exact** deliverable — confirming the engine pipeline is intact after the revamp.

## Known limitations / follow-ups (post-revamp, honest)
- ACCESS/SUBAGENT reverse-harness grants remain observation-only (the autonomous path narrates them; full provisioning is future work).
- The deeper Insights metric-store unification (one accessor behind flywheel/learning/memory metrics) is still three stores under the hood.
- `FileVault` doesn't implement the episodic `query_session_summaries` API (a non-fatal log on file-backed vaults; the sqlite/postgres backends are unaffected).

## Migration
No vault migration. Additive and backward-compatible — `/army` and the other old URLs redirect, `build_console_page` is kept as an alias, and agent/engine behavior is unchanged. All v0.9.0–v0.9.11 features untouched.
