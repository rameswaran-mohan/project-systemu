# v0.9.33 — harness-runtime reliability: live v2 tools, bounded delegation, budget-aware runs

Five validated harness-runtime fixes from an internal bug report, each
implemented test-first, then put through a multi-agent adversarial review
(every finding faced an independent refutation panel). Five of six review
dimensions came back clean; the one confirmed gap (an artifact-reporting
omission) is fixed and noted below. No behavior outside the harness path
changes.

## What changed

1. **Code-registered ("v2") built-in tools now actually run.** Tools like
   `write_file`, `read_file`, `search_files`, `spawn_subagent` and
   `mixture_of_agents` were advertised to the agent but were undispatchable —
   the loop's tool handler only resolved old-style vault tools, so a call to a
   v2 tool fell through to "tool not found." The handler now short-circuits
   v2 tools through the v2 dispatcher (preserving the dry-run short-circuit and
   the v0.9.32 per-command shell-approval gate, which are unaffected because the
   shell tools are not v2-registered).

2. **The per-run harness request cap is enforced again, and the
   subagent-delegation cascade is bounded.** A Shadow can now make at most
   `max_requests_per_run` capability requests per run (the count is threaded
   into the arbiter and fires at *exactly* the limit), and that count now
   **survives suspend/resume** so a parked-then-resumed run keeps counting.
   Sub-agent delegation is bounded on three axes: the depth guard uses the
   **actual** nesting depth (not just the model's self-reported depth), spawned
   children run with delegation **disabled** (so they cannot recurse into
   another fleet), and a fleet's result is delivered as a **terminal**
   observation (synthesize + finish, never re-delegate). Together these close
   the runaway-cascade paths.

3. **The agent can see its iteration budget.** Each decision step now receives
   `iteration`, `iter_budget` (the live ceiling — a granted compute extension
   raises it mid-run) and `iterations_remaining`, plus a low-budget nudge when
   the budget is nearly spent and work is still pending. A new decision rule
   tells the agent to wind down and consolidate toward `COMPLETE` rather than
   looping blindly to the ceiling.

4. **The ACCESS capability grant tells the truth.** On the local single-owner
   backend a granted "access" lease is **advisory** — no sandbox boundary is
   actually enforced — so the grant message no longer claims a boundary
   authorizes the operation; it records the advisory lease and tells the agent
   to proceed with its existing tools. The dead sandbox-policy "apply patch"
   plumbing (built by the Governor, forwarded by the scheduler, reconstructed by
   the runtime, and consumed by nothing) was removed end-to-end.

5. **v2-written files are recorded as produced artifacts.** Adversarial review
   caught that a v2 `write_file` deliverable was omitted from a run's
   `files_produced` whenever the working directory differed from the output
   directory (the normal case), because the file tool didn't echo the resolved
   path the artifact collector needs. The file was always written correctly and
   goal verification was unaffected, but session summaries and episodic recall
   under-reported the deliverable. The file tool now echoes its resolved path,
   matching how the old vault file tool already behaved.

## Known limits (by design, this release)

- The harness request cap and depth guard are **opt-in** via the intent engine;
  with the engine off, behavior is unchanged.
- A Shadow provisioned with *only* v2 tools (and no deployed vault tool) is still
  gated by the pre-existing "no deployed tools" precondition — a separate
  follow-up. In normal deployments Shadows carry seed vault tools, so this does
  not arise.
