# v0.9.74 — R-A13b-2ii-a: money-move & message-sending effect classification

The final data slice of the external-verification net's shadow-measurement stage. Until now the net
could recognize a network write but not *what kind* — so in shadow mode nearly every effectful tool
was measured as a potential money-move. This release adds a curated signal map so the system
recognizes when a tool actually touches a **payment/transfer** capability (Stripe, PayPal, Plaid, …)
or a **message-sending** one (Twilio, Slack, email, …), and classifies the effect accordingly. The
net stays **off**; this is still shadow-mode measurement that changes no run outcome.

## What ships

- **Effects are classified by their real type.** A curated, reviewable map of well-known
  payment/transfer and messaging SDKs, API hosts, and call patterns lets the shadow meter bucket each
  effect as *money-move*, *send-message*, or a plain network write — instead of a single catch-all.
  The shadow park-surface report and readiness verdict now reflect the true mix of effect types.
- **Fail-closed on money — a money-move is never quietly downgraded.** Any money signal, on any axis
  (import, host, or call), *always adds* the money-move classification; an ambiguous effect stays
  unclassified (the strictest treatment) rather than being labeled as something safer. This preserves
  the guarantee that a money-moving effect can only be confirmed by an independent, host-pinned,
  provably-fresh read-back — it can never be self-confirmed by the tool.
- **More precise measurement.** A tool with several effects is now reported under its most significant
  one, so a message-sending tool no longer masquerades as a money-move in the report.

## Compatibility & scope

- **Read/shadow only, no behavior change:** classification feeds the shadow meter and the effect
  gate's existing stamp set; no run outcome changes in any mode, and the net remains off. The full
  suite (5426 tests) is green.
- The curated signal map is kept comprehensive over time; a money-move via a not-yet-curated provider
  under a non-money-worded objective is a known, bounded limitation the shadow report surfaces.
