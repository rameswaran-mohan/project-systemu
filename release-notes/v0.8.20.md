# v0.8.20 — bugfix: `fetch_json` sends a proper User-Agent so free location/data APIs work out of the box

A focused patch release. The built-in `fetch_json` tool now sends a descriptive `User-Agent` (and a JSON
`Accept` header), so the free geo/data APIs the agent already reaches for stop rejecting its requests.
Behavior-only change to one seed tool — no API change, no new dependencies, no migration.

## The bug

`fetch_json` left the `User-Agent` unset, so requests sent its default `python-requests/x`. Several free
APIs the agent commonly reaches for refuse that user agent outright:

- **Nominatim returns 403** — it enforces the OSM usage policy, which requires a meaningful, identifying
  User-Agent and blocks the bare `python-requests` default.
- **Overpass returns 406** — it rejects the request before any data comes back.

Because these are exactly the endpoints behind the free "find nearby places / restaurants / salons" path,
those tasks failed with no usable data source — the agent had no working way to answer a location query
without a paid key.

## The fix

`fetch_json` now defaults a descriptive `User-Agent` and `Accept: application/json` when the caller does
not supply them. Caller-provided headers always win, so anything that already set its own headers is
unaffected. This unblocks the free geo path the agent already tries — no new code path, just sane defaults
on the existing one.

Confirmed by a live POC: the same Nominatim request that returned **403** now returns **200**, and the
Overpass **406** rejection is cleared.

## Compatibility

Behavior-only change to a single seed tool. No API change, no new dependencies, and no migration.

> **Note for existing vaults:** vaults keep their *copied* tool until they are re-seeded. A fresh
> `sharing_on init` or a vault reset picks up the new tool; a package upgrade alone does **not** rewrite
> tools that are already seeded into an existing vault.

## Files changed

Modified:
- `systemu/vault/tools/implementations/fetch_json.py` — default `User-Agent` + `Accept: application/json`
  via `setdefault` (caller headers win)
- `tests/test_v0_8_20_fetch_json_headers.py` — covers the defaulting, the caller-override precedence, and
  the JSON `Accept` default
- `systemu/__init__.py`, `sharing_on/__init__.py`, `pyproject.toml` — 0.8.19 → 0.8.20

## Migration

```
pip install --upgrade systemu==0.8.20
```

No new dependencies and no configuration changes. To pick up the fixed tool in an existing vault, re-seed
it (`sharing_on init` into a fresh vault, or a vault reset); a package upgrade alone does not rewrite
already-seeded vault tools.
