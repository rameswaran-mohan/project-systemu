# v0.9.8 — Keyless Web Access + Process-Objective Verification + Autonomous Coach

**Release date:** 2026-06-09
**Headline:** Multi-step "near me" web tasks now **complete end-to-end**. The agent's web tooling no longer 403s on a raw socket — it goes through a layered, **keyless** stack (Jina Reader + Jina-on-DuckDuckGo + OpenStreetMap) with a structured `find_places` tool; intermediate "obtain/determine/search" objectives are now **credited from tool-action audit evidence** (the keystone) instead of being falsely rejected into a stall loop; the runtime self-steers on stalls before bothering the operator and survives malformed LLM output instead of crashing.

## What landed

### Keyless layered web access (`systemu/runtime/web_access.py`)
Replaces raw `requests.get` (which Cloudflare/anti-bot 403s) with a layered, **no-API-key** stack, proven against live sites in Phase-0 probing:
- **`read_url`** — Jina Reader (`r.jina.ai`, bypasses most 403s) → raw GET → capped Chromium-stealth render for JS/hard pages.
- **`search_web`** — Jina-on-DuckDuckGo (renders DDG's results page keyless) → raw DDG-lite fallback; returns parsed `{title, url, snippet}`.
- **`find_places`** — new vault tool: OSM Nominatim geocode + Overpass POIs → **named local businesses** for "X near me" (the capability the agent was missing).
- The existing `web_extract`/`web_read`/`web_search` tools delegate to this layer under `SYSTEMU_WEB_STACK_V2` (default on; legacy path retained behind the flag, re-read at call time).

**Guardrails (built in, not bolted on):** per-host rate-limiter (honors Jina ~20 RPM, Nominatim 1 req/s), TTL result cache, descriptive User-Agent, **OSM/ODbL attribution** surfaced in `find_places` output, Chromium concurrency cap.

**Scope honesty:** the keyless stack is sized for **personal / single-user / light** use. It relies on free public instances (Jina, DuckDuckGo, Overpass/Nominatim) whose terms and rate limits make them **unsuitable for multi-tenant / scaled** deployment. An optional, flag-switchable key/self-host path (Brave/Tavily/SearXNG) is scaffolded **off by default** for anyone who outgrows personal scale. Keyless general search is inherently ToS-gray (all engines forbid scraping); this is a deliberate trade for "no keys."

### Autonomous mid-run coach (`systemu/runtime/coach.py`)
On a no-progress stall, the runtime now generates a **specific corrective steer** via the supervisor/governor LLM (e.g. "use `find_places` for local lookups", "you have enough — write the file") and injects it to continue, escalating to a human only after `auto_coach_max_steers` self-steers fail. Fail-safe: any LLM error → no steer → falls through to the existing operator path. Flags: `SYSTEMU_AUTO_COACH`, `SYSTEMU_AUTO_COACH_MAX_STEERS`.

### Process-objective verification — the keystone (`shadow_runtime.py` + verifier prompt)
The per-objective verifier only sees the **StateDelta** (files + audit entries + chat reply). The runtime never wrote an audit entry for ordinary tool calls, so an **intermediate "obtain / determine / search" objective** (e.g. *"location obtained and stored"*) had **zero durable evidence** and was falsely rejected — stalling every multi-step web task in a rejection loop. Now:
- The runtime writes **one compact, length-capped audit row for every tool call that succeeds** (action + truncated params + objective id + success), so `compute_delta` surfaces it to the verifier. Best-effort — it never breaks the run.
- The verifier prompt makes **audit / tool-action evidence first-class** for obtaining/determining/searching objectives (no file demanded unless the criteria name one). File-deliverable and final objectives stay strict (they must show the concrete artifact).

### Extraction robustness (minimal Phase 3)
- An **empty extraction** (LLM returns no tools/skills) no longer silently aborts the run — it **falls back to the full curated toolset** and proceeds (the agentic loop selects tools at runtime).
- Malformed LLM output (a bare string/list where a dict was expected) no longer crashes with `'str' object has no attribute 'get'`.

### Reliability hardening (B1–B7) — found by live-validating diverse use cases
Re-running diverse "near me" / research tasks (NSE pick, Tamil rasi, burrito, weather)
exposed that the keystone alone wasn't enough; five further fixes make well-posed
file-deliverable web tasks actually complete end-to-end:
- **B1 — output-path redirect on the REAL dispatch path.** The v0.9.7 redirect lived
  only in `ToolSandbox.execute()`, but the agentic loop dispatches via `execute_tool()`
  — so it **never fired in real runs**. A model slip like `file_write(path="/tmp/x.txt")`
  escaped output_dir, the verifier never saw the deliverable, and the run looped to
  MAX_ITERATIONS. Now applied in `execute_tool` (NSE went partial→success).
- **B4 — verifier judges by EFFECT, not exact tool name** (a `fetch_json` to a geo API
  satisfies "obtain location" even if the hint named `geolocation.get`).
- **B5 — verifier failures SOFT-PASS** instead of falsely rejecting (the goal-level
  verifier remains the deliverable backstop).
- **B7 — `max_tokens` for the structured LLM calls** (verifier/coach/goal-verifier) raised
  200/300 → 1500. The reasoning models emit visible chain-of-thought that consumed the
  tiny budgets **before** any JSON was produced, so the **coach was dead** (`→ no steer`
  every call) and the verifiers mislabelled correct work. This was the systemic root
  behind most "partial" outcomes. (burrito went partial→success once the verifier could
  confirm the already-correct file.)
- **B2 — research-loop convergence steer**: after N consecutive read-only research calls
  with no deliverable, force a "stop searching, write now" steer (the keystone's audit
  credit kept resetting the stall counter, so the stall path missed this loop).
- **B3 — keyless reliability**: `find_places` retries the Overpass hosts a 2nd pass with
  backoff (mirror list grown to 4), and Nominatim geocode retries once — mitigates the
  free-Overpass 504/timeout failures.
- **B6 — verifier defaults to Tier-3** (free, sufficient for a yes/no verdict).

### Extraction robustness (minimal Phase 3)
- An **empty extraction** (LLM returns no tools/skills) no longer silently aborts the run — it **falls back to the full curated toolset** and proceeds (the agentic loop selects tools at runtime).
- Malformed LLM output (a bare string/list where a dict was expected) no longer crashes with `'str' object has no attribute 'get'`.

### Also
- `dep_approvals` degrades gracefully when `SYSTEMU_DATABASE_URL` points at an unreachable DB (postgres without psycopg2) instead of crashing the tool dependency-check path.

## Operator knobs (new)
| Env var | Default | Purpose |
|---|---|---|
| `SYSTEMU_WEB_STACK_V2` | `true` | Keyless web layer (set `false` for the legacy raw path) |
| `SYSTEMU_WEB_READER_BACKEND` / `SYSTEMU_WEB_SEARCH_BACKEND` | `auto` | Override reader/search backend |
| `SYSTEMU_WEB_CACHE_TTL` | `900` | Web result cache TTL (seconds) |

| `SYSTEMU_RESEARCH_LOOP_THRESHOLD` / `SYSTEMU_RESEARCH_LOOP_MAX_STEERS` | `5` / `2` | B2: force a deliverable after N read-only research calls |
| `SYSTEMU_VERIFIER_TIER` | `3` | B6: tier for the per-objective verifier (was 1) |
| `SYSTEMU_BRAVE_API_KEY` / `SYSTEMU_TAVILY_API_KEY` / `SYSTEMU_SEARXNG_URL` | `""` | Optional scaled search backends (off) |

## Known limitations / follow-ups
- **Sparse-task last-mile convergence** — on ill-posed/sparse-data tasks (e.g. monthly horoscope), the agent can find the right answer but file it under the wrong name / run out of iterations. The B2 steer + working coach (B7) mitigate but don't fully close this.
- **Keyless latency & availability** — the polite rate-limiter + free public endpoints (Jina/Overpass/Nominatim) make multi-call tasks slow (~3–18 min) and occasionally 504/timeout; B3 retries help but keyless is personal/light-use only.
- **Verifier transient** — a rare `'NoneType' object is not subscriptable` in the verifier LLM path is caught by the B5 soft-pass (non-fatal); root-cause trace is a follow-up.
- **Cost guard** (per-objective token/iteration ceiling) — deferred; `MAX_ITERATIONS` still bounds runs.

## Migration
- No vault migration. The keyless web stack is on by default; set `SYSTEMU_WEB_STACK_V2=false` for the exact legacy behavior. All v0.9.0–v0.9.7.1 features untouched.
