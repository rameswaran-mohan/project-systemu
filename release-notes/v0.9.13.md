# v0.9.13 — Design fidelity, first-run workflow success, security hardening

Three arcs in one release: the dashboard finally matches the agreed design
boards, a first task now succeeds (or tells you exactly why it can't), and
LLM-generated tool code lost its free run of the daemon process.

## Design fidelity (the agreed boards, actually shipped)

- **Material line icons everywhere** — nav, page headers, buttons. The emoji
  icon set (🔧 📜 🧠 …) is gone from the web UI (313 glyphs across 22 files);
  Skills and Memory no longer share the same brain.
- **The Refined Midnight Indigo identity** — signature radial gradient
  background, Inter + JetBrains Mono, flattened Material chrome (no ripple
  shadows / forced uppercase), board-spec status pills and table headers.
- **Readable buttons** — flat Quasar buttons inherited a blue label on tinted
  backgrounds (contrast ~1.2: the "+ New" / "Record" text was effectively
  invisible). Per-variant text colors fix it; audited Home/Tools/Shadows to
  0 low-contrast elements.
- **Tools toolbar** — live search + status filter over the registry.
- Off-palette colors (Tailwind light-red/blue banners, raw `#ef4444` reds)
  → design tokens; deprecated-route links (`/flywheel`, `/memory`,
  `/notifications`, `/systemu-chat`) now go straight to their live targets.

## Workflow success (your first task should not fail silently)

- **Blocked tasks are actionable** — when the readiness gate parks a task on
  not-yet-enabled tools, it now posts an Inbox card naming the blockers;
  **Enable & run** executes the real Gate-3 enable per tool (the dry-run rule
  stays enforced; never auto-granted, even under Bypass). The workflow detail
  page shows a blocked-on-tools banner, and Work rows tint warn.
- **Statuses tell the truth** — synchronously-executed tasks now mark their
  activity COMPLETED (previously stuck at "assigned" forever); the workflow
  detail page shows the **Artifacts folder** where deliverables land.
- **Headless contexts can't hang or lie** — `SYSTEMU_NON_INTERACTIVE` now
  covers the destructive-action confirm gate (one shared detector across all
  prompt paths), and a headless auto-deny is logged to the event stream
  instead of silently degrading the run.
- **Episodic memory works on the default backend** — the session-summary
  surface (append/query/search) was never proxied through FileVault or
  ParallelVault; ParallelVault was also missing the decisions surface
  (the Inbox couldn't read pending decisions in parallel mode).
- **Tools resolve from any directory** — implementation paths are anchored to
  the vault at construction instead of floating with the process CWD.

## Security hardening

- **Forged tool code runs out-of-process by default.** The in-process fast
  path (a direct import inside the daemon — full process privileges) is now
  reserved for built-in tools; LLM-generated tools execute via the subprocess
  sandbox unless the operator explicitly marks them `trusted_inprocess`.
- **The Gate-2 risk scan is AST-based and honestly framed.** The old
  substring scan was trivially bypassed (`getattr(builtins, 'ev'+'al')`);
  the new scan resolves call names, `shell=True` keywords, getattr
  obfuscation, and string payloads — and is labelled an *advisory* scan, with
  no more green-check "no dangerous patterns" false confidence.
- **AUTO_FORGE needs a second factor.** `SYSTEMU_AUTO_FORGE_TOOLS=true` alone
  no longer disables the three tool gates — it also requires
  `SYSTEMU_AUTO_FORGE_ACK=I_UNDERSTAND_ALL_GATES_ARE_BYPASSED`.
- **Piercing the safety floor is visible.** `no_floor` or an `allow` override
  on a floor gate type (dep installs, destructive recovery) now shows a
  persistent warning banner on the Inbox and Settings.

## Verification

63 new tests across the release; the full suite (2,844 passed) shows zero
new failures versus the pre-release baseline; UI-style lint debt reduced.
