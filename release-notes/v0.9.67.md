# v0.9.67 — R-A11a: the Reference Resolver (§5.4)

**"My resume" now resolves to the right file — scored, confined, and never silently.** When an
objective's required file parameter isn't supplied literally ("summarize my resume", "open the
sheet"), the requirement binder used to blind-bind the *first* salient file in a granted root —
a wrong-file guess dressed up as a confident bind. This release replaces that with a real
reference resolver (spec §5.4): it scores the objective's own words against the surveyed files,
confines every candidate to a granted root, and surfaces the match as a one-click operator
confirm — never a silent bind.

## What ships

- **A scored reference resolver** (`runtime/reference_resolver.py`, `runtime/reference_synonyms.py`).
  It extracts reference tokens from the objective goal (+ success criteria), synonym-expands them
  to extension hints ("sheet" → `.xlsx/.csv`, "deck" → `.pptx`), and scores every salient file by
  `containment-overlap(name, reference) + extension-match + recency` — reading only the
  situational-inventory salient snapshot the survey already captured (**no disk walk, no new
  durable writer**).
- **Confinement is enforced.** Every candidate is re-gated through the GrantedRoots store; the
  primary boundary (only granted roots are ever surveyed) means an out-of-root path can't even
  become a candidate (§5.4 AC-a). A resolved file is clamped `content_derived` regardless of
  score, so it is **always** an operator confirm — a high score never launders a file value into
  a trusted, silent bind (IMPL-5).
- **No false confidence.** A genuinely ambiguous reference (two files scoring within an epsilon)
  is capped below the confidence threshold so it can never auto-accept — the operator confirms or
  corrects (§5.4 AC-b). A reference that matches nothing degrades to an honest ask-for-path, and
  any resolver error degrades to a best-effort gap that never empties the objective's plan
  (fail-safe).

## Compatibility & scope

- **Behavior change (net-positive):** a path parameter whose objective doesn't lexically name any
  surveyed file now surfaces an honest ask instead of binding a wrong file. Because file binds
  were already `content_derived` (always asked), this reshapes the ask rather than adding one —
  the full suite (5125 tests) confirms no regression.
- **Scope (R-A11 split).** This is R-A11 **Module A** (Reference Resolver). Module B — the
  discovery ladder (`find_tools`, discovery-before-forge) and the PLAN-6 SSRF-guard dedupe — lands
  as **R-A11b**. The structured multi-candidate disambiguation *card* (vs. today's top-candidate
  confirm) is a surface concern deferred to R-A13a.
