# v0.8.6 — Execute queue + Schedule + EventBus bridge

v0.8.5's analysis concluded that the operator `▶ Execute` button on `/army`
needs bounded concurrency, scheduling, and real-time event visibility — but
must NOT route through `Supervisor.submit` (which is reserved for
autonomous pipeline flows). Conflating the two collapses the distinction
between human-initiated interactive work and pipeline-initiated autonomous
work, which is an anti-pattern for AI PA agents.

v0.8.6 ships four cooperating subsystems anchored on the unchanged Execute
subprocess model.

## What's new

### 1. JobManager execute queue
- New `JobStatus.QUEUED`. Execute clicks past the concurrency cap go into a
  pending deque and a background dispatcher promotes them as slots free.
- Env-configurable: `JOBMANAGER_MAX_CONCURRENT_EXECUTE` (default 3),
  `JOBMANAGER_MAX_EXECUTE_QUEUE_DEPTH` (default 50).
- Per-job dedup: `f"execute:{shadow_id}:{scroll_id}"` — double-clicks return
  the existing job instead of spawning twice.
- `cancel_queued(job_id)` lets operators drop a queued job before it spawns.

### 2. Schedule registry
- Operator-created schedules persist as `vault/schedules/schedule_<id>.json`.
- One-time and recurring (interval ≥ 5 min) supported.
- Skip-missed semantics: if the dashboard was down past a fire time, the
  schedule fires once on restart and resumes — no burst-fire backlog.

### 3. Scheduled execute dispatcher
- New APScheduler job `_scheduled_execute_job` runs every minute.
- Finds due schedules; dispatches each through JobManager with the same
  dedup key as a manual click; advances `next_fire_at` (recurring) or marks
  COMPLETED (one-time).
- Logs each fire and skip reason.

### 4. EventBus bridge for Execute subprocesses
- Subprocess installs a writer that mirrors every EventBus event to
  `vault/manual_events.jsonl` (atomic append-per-line).
- Dashboard runs a `ManualEventBridge` tailer in a background thread; new
  lines are parsed, tagged with `context.origin = "manual_execute"`, and
  republished onto the dashboard's in-process EventBus.
- One-deep file rotation at 10 MB.
- `/insights → Events` tab gains an All/System/Manual filter dropdown.

### UI additions on /army
- **Recent Execute Jobs** panel: three columns (Queued / Running /
  Completed), up to 30 rows total, per-row Cancel button, queue position
  numbers, 2-second refresh.
- **Schedule** button next to Execute on every shadow card → opens a
  one-time/recurring dialog with scroll picker, dry-run toggle, datetime or
  interval inputs.
- **Active Schedules** panel: list of scheduled fires with relative time
  ("in 23 min") and per-row Cancel.

## Backward compatibility

- `JobManager.start_job` gains an optional `dedup_key=""` parameter. Old
  callers continue to work.
- `Job.dedup_key` is a new field with default `""`.
- New env vars are unset by default — defaults match prior behavior.
- No vault schema changes to existing entry types. `vault/schedules/` is
  a new directory; the dashboard creates it on first schedule.

## Migration

`pip install --upgrade systemu`. No `.env` changes required. The
`vault/schedules/` directory is created automatically on first Schedule
creation; the `vault/manual_events.jsonl` file is created automatically on
first dashboard boot.
