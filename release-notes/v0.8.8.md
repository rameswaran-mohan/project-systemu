# v0.8.8 — Console revamp

Consolidates the operator's most-used surfaces into a single Console screen and
adds Edit deep-links from list pages into the existing Workshop editors.

## What's new

### Console (was Overview)
- Renamed Overview → **Console**, moved to a standalone sidebar item at the very
  top, above the "Run" group.
- **6 clickable stat tiles** — clicking Scrolls / Shadows / Tools / Skills /
  Activities / Evolutions navigates straight to that list page.
- **Pending Actions mini-pane** at the top — same decision cards as
  /insights → Pending Actions, resolvable inline.
- **Two live event panes**: Supervisor live events (in-memory EventBus stream,
  left) and the Insights event log (file-tail, right), both color-coded and
  scrollable.
- **Quick Actions** moved into a compact header row (Record / Evolve /
  Notifications / Forge / Restart Workers).
- The five analytics cards (Learning Curves / Memory / Skills Snapshot /
  Pending Tools / Tool Dependencies) remain at the bottom, collapsed.
- The Workflow Pipeline card and the Recent Activity feed are removed from the
  Console (the event panes supersede them).

### Edit deep-links
- `/scrolls`, `/army`, `/tools`, `/skills` rows now have a **✏️ Edit** button.
- Edit navigates to `/workshop?type=X&id=Y`; Workshop opens the matching tab and
  auto-opens that entity's editor (shadow/tool/skill) or pre-selects the scroll
  for the LLM-rebuild flow.
- No new editors — Workshop already had full Scrolls/Shadows/Tools/Skills
  editing; this is deep-link wiring.

## Backward compatibility
- `/` still resolves; it now renders the Console.
- `build_overview_page` is re-exported from `pages/overview.py` (now a shim
  pointing at `build_console_page`), so any external importer keeps working.
- All existing routes resolve unchanged.
