# v0.9.72 — R-A13b-2i: the external-verification evidence pipeline (independent read-back)

Stage 2 of turning the external-verification net on continues. v0.9.71 added the shadow **meter**
that measures how the net would behave; this release adds the **evidence pipeline** it measures —
an independent, safety-hardened read-back path — so the shadow measurement can record real
confirmations instead of parking everything. The net stays **off**; this is still shadow-mode
plumbing that changes no run outcome.

## What ships

- **An independent read-back channel.** When a tool participates in external verification, the
  runtime can now re-read the effect over an independent, credential-less HTTPS connection to confirm
  it actually landed — rather than taking the tool's word for it. This is what lets the shadow meter
  record a genuine "would-confirm" for a real effect.
- **Money-move effects can only be confirmed independently.** A money-moving effect is now confirmed
  **only** by that independent, host-pinned, provably-fresh read-back — never by the tool's own
  reported receipt, an inline token echo, or any advisory signal. A tool cannot self-confirm that it
  moved money.
- **Anti-replay freshness, bound to the confirmed resource.** Confirmation requires proof that the
  effect's one-time token did not already exist before the run (a stale/replayed receipt cannot
  confirm), and that proof is bound to the exact resource being confirmed.
- **The read-back channel is SSRF-hardened.** The independent reader refuses any non-public or
  non-HTTPS target — including private, loopback, link-local, and cloud-metadata addresses, and IPv6
  literals that embed an internal IPv4 — before it ever connects, and it never follows a redirect.

## Compatibility & scope

- **No behavior change with the net off (the default):** nothing is stamped, credited, asked, or
  parked; run outcomes are unchanged. The independent reader is only wired in when the net is in an
  armed mode, and it changes no run's result. The full suite (5263 tests) is green.
- Populating which effects count as money-move / message-sending, and the operator-facing shadow
  report, are the next steps.
