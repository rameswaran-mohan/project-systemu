# v0.9.30 — welcome starters actually open Chat (no more "just refreshes")

A first-run UX fix. On the welcome page, clicking a **starter use-case**
appeared to "just refresh the page" instead of opening Chat with the prompt
pre-filled.

## Root cause

The starter click bare-navigated to `/chat?prefill=<prompt>`, but the W11.4
onboarding gate (`_redirect_to_welcome_if_needed`) redirects **every** route
back to `/welcome` until onboarding is complete (`key_present` +
`profile_present`). Since clicking a starter didn't *save* anything, the gate
bounced it straight back — dropping the prefill. The starters were effectively
dead during setup.

## Fix

Extracted `finalize_onboarding()` — validate (API key + name required) and
persist the preset + profile, returning `(ok, message)` — and shared it between
the **Finish** button and the step-4 starters. Clicking a starter now finishes
setup first (which sets `profile_present`, satisfying the gate — the redirect
requires only key + profile, never the tour), **then** navigates to
`/chat?prefill=…`, which renders normally. A validation failure (missing key or
name) notifies and stays put, exactly like Finish.

So a starter click now does the intuitive thing: completes onboarding *and*
lands your first task in Chat, ready to Run.

## Verification

New tests: `finalize_onboarding` blocks without key/name (persisting nothing),
finalizes + saves the profile on valid input, and a wiring guard that the
starter click routes through the finalize path rather than a bare navigate.
Full suite green (3366 passed).
