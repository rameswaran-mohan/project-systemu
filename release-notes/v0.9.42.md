# v0.9.42 — dashboard UI polish (duplicate titles, fluid panels, Live unread dots, fixed "Needs you")

Pre-launch dashboard corrections. No runtime/behaviour change — presentation only.

## Removed duplicate page titles

Home, Work, Scrolls, and Inbox each rendered their own page title on top of the
global header, so the title showed twice (on Home, "Home" sat redundantly beside
the Record button). Each page's local title is dropped; the thematic sub-label
below it is kept, and Home's action buttons are re-aligned to the right.

## Fluid side-panel widths

The left sidebar and right rail now scale with the viewport instead of being
pinned:

- Left sidebar: `220px` → `clamp(160px, 13vw, 200px)`.
- Right rail: `300px` → `clamp(300px, 22vw, 360px)`.

The existing responsive behaviour (collapse below 768px, hide the rail below
1100px) is unchanged.

## Live pane — chevron on the left + per-session unread dots

Streamed Live event rows now place the expand chevron on the **left** (Quasar
`switch-toggle-side`) and carry a small leading unread dot: a solid accent dot
when the entry is new, hollowing to an empty ring once the operator expands or
clicks it. Read-state is in-memory (per session). `stateful_expansion` gains an
opt-in `read_state` path; the default call site is unchanged.

## "Needs you" — fixed height, stable width, thin scrollbar

As gates pile up, the "Needs you" inbox glance was growing and squeezing the
Live pane beneath it. It is now a fixed `220px` height with its own scroll:

- Content width stays constant when the scrollbar appears (reserved gutter).
- A thin, near-invisible scrollbar (brand-tinted on hover).
- Compact Approve/Deny buttons so a queued gate fits in less vertical space.
