# v0.8.22.3 — Re-ship of v0.8.22.2 (no code changes)

Ship-mechanics patch. **No code changes — identical to v0.8.22.2.**

The v0.8.22.2 tag was force-moved on origin after the initial sync proved
incomplete, and GitHub Actions silently skipped the second `push: tags`
trigger so the wheel was never built. Rather than disturb the tag history
further, v0.8.22.3 re-ships the same code under a fresh tag, which triggers
`publish.yml` cleanly.

## What's in this release (unchanged from v0.8.22.2)
- `fix(web_extract)` — LLM-friendly `fields=[...]` interface + heuristic fallback (root cause of the live burrito-task failure).
- `fix(resume)` — cross-process reconciler for CLI-resolved stuck decisions (15s daemon poll; persisted `resume_dispatched` flag).
- `fix(capture_to_skill)` — fresh-extract branch uses `Activity.required_skill_ids` instead of treating the Activity as a dict.

Full details: see [release-notes/v0.8.22.2.md](v0.8.22.2.md).

## Upgrade
`pip install -U systemu`. No vault migration. Restart the daemon to pick up the new resume reconciler + web_extract interface. Anyone who somehow installed v0.8.22.2 (none should have — it never reached PyPI) gets bit-identical code.
