# v0.9.19 — Knows who it works for, reaches beyond the desk

Two waves in one release. Wave 9 gives systemu an identity and connections:
a first-run that collects who you are, a profile that every lane actually
uses, and MCP servers as first-class tool sources. Wave 10 extends its reach
and accountability: needs-you moments land on your phone, automated trust
decisions leave an audit trail, and a golden-task gate pins outcome quality
in CI.

## It finally asks who you are

Fresh installs used to land on an empty dashboard with a warning banner —
the profile model existed since v0.9.0, but nothing ever collected it, which
is why a real run once guessed the operator's location by IP. New installs
now land on **/welcome**: a four-step wizard covering API-key status (status
only — keys are never typed into the browser), model preset, your office
profile (name, location, timezone, output folder, plus role/organization and
a persona pick: personal / freelance / solo founder / small business /
enterprise), and three safe starter prompts that land pre-filled in chat.
Skip is honored permanently — "later" never nags.

## …and every lane knows it

One profile-context composer now feeds the refiner, the runtime, **and the
quick lane** — the fastest path is no longer the most identity-blind one.
"Find a spa near me" now means near *your* city, in *your* timezone, saving
to *your* output folder. Clarify-first prompting also got office-grade:
before refining a workflow, the refiner asks for the essentials a colleague
would (recipient, source file, date range, amount, account) instead of
guessing.

## Plug in your tools (MCP)

Settings → Connections now manages **MCP servers**: add a server by URL,
discover its tools, and enable them per-tool — everything is **off by
default**, and a connector can never shadow or hijack a built-in vault tool
(vault tools always win on name collisions). Enabled MCP tools become
callable from the quick lane through the same honest-results contract as
local tools.

## It can reach you on your phone

The Telegram gateway (token + strict user-ID allowlist via environment
variables only) is now wired into the daemon: **needs-you decisions and task
outcomes push to your phone**, with rate limits so approvals always get
through but chatter doesn't. A `/status` command answers from anywhere with
the same needs-you count and recent-task list as the dashboard's Status
button. Settings shows a status-only Telegram section — tokens are never
entered in the browser.

## Trust you can audit

When a gate-mode policy auto-allows a step, that decision used to vanish —
the run proceeded and nothing recorded why. Auto-allowed gates now write a
**born-resolved decision row** tagged `auto-policy`, visible in the Inbox
history with a pill — without ringing the needs-you badge or your phone.
Every grant, human or policy, is now in one ledger.

## Quality, pinned

- **Golden-task gate:** six keyless end-to-end tasks (research, file
  production, multi-step, clarify, failure-honesty, profile use) assert on
  *outcomes*, not mocks, and run in the normal suite.
- **Plain language beside the lore:** Scrolls, Shadow Army, and Activities
  pages now carry one-line plain-English sublabels ("Workflows — captured
  task definitions you can run again") so a non-technical operator is never
  lost in the theme.

## Verification

65 new tests across nine slices, all runnable without an API key (stub MCP
server over loopback HTTP, scripted LLMs, fake Telegram transport). Full
suite: 3,081 passed, 0 failures.
