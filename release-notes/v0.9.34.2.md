# v0.9.34.2 — executor honesty disposition (no fabricated results)

A one-change patch on top of v0.9.34.1. It hardens the agent's instructions
against fabricating its own computed values.

## What changed

The executor prompt already forbade fabricating **tool outputs** ("Never
fabricate tool outputs"; "Reasoning alone cannot satisfy a `success_criteria`"),
but it never told the agent not to fabricate its **own computed values**. So when
a goal asks for an exact result the model wrongly believes it can produce — a
hash, a checksum, exact arithmetic — it would emit a confident, plausible, wrong
value instead of acquiring a real capability. (Observed: frontier models
fabricating a SHA-256 rather than forging a hashing tool, even with the
`REQUEST_HARNESS` affordance present.)

This adds a short, generic **"How you work"** disposition near the top of the
executor prompt: use the capabilities you actually have, acquire the ones you
lack, and never invent or approximate a result you couldn't defend if asked to
show your work — an unverifiable answer that merely looks right is a failure. The
disposition deliberately names no task, no operation type (no "hash"), and no
specific action (no "`REQUEST_HARNESS kind=…`"): it sets a disposition and leaves
the agent to recognize its own limits. The model's overconfidence isn't ours to
fix, but the agent's instructions are — and this closes a genuine
instruction-quality gap that applies to any deployment.

Prompt-only change (`systemu/prompts/execute_step.md`) plus a prompt-content
test. No behavioural code paths change.
