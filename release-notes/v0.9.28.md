# v0.9.28 — the quick lane stops throwing away answers it already found

Root-cause fix for the deeper failure behind v0.9.27. A *"find me best punjabi
restaurant near me"* chat task ran 12 steps, **found a real, well-reviewed,
genuinely-near restaurant at step 7**, then returned `failed` with nothing —
because the loop's only path to an answer was the model voluntarily emitting
`ANSWER`, and on budget exhaustion it discarded everything it had gathered.

The principle: **stop delegating guarantees to soft prompt instructions; the
machine enforces them.** The prompt already said "answer when you can / don't
repeat failures / reserve the last turn" — the model ignored all three.

## What changed (in `quick_task.py`)

1. **Harvest on every exit.** On any non-`ANSWER` termination (budget,
   wall-clock, fail-cap, malformed, mid-loop LLM error), the lane now runs a
   grounding-only, cheap-tier synthesis from the gathered observations and
   returns it as **`partial`** — never a fake `success`, with the failure
   reason retained. **Gated on usable evidence:** if nothing usable was
   gathered (all tools errored, or a "success" had an empty payload), it stays
   an honest `failed`. No inventing from nothing.

2. **Block the repeat.** A `(tool, params)` call that already failed is refused
   with an actionable nudge instead of re-executed — killing the
   step-1-equals-step-12 loop from the transcript. Blocked retries still count
   toward the same-tool cap so an unchanged retry can't thrash to the budget.

3. **Plan-first, adaptive.** A non-trivial task decomposes into a short todo
   (`PLAN`) before acting, and the plan rides in every later turn so execution
   follows it. Trivial asks pay no planning tax — they answer on turn 1. The
   payload now carries `iterations_left` / `final_turn`, and the last action
   turn is told to answer with what it has.

4. **Operator location used verbatim.** The profile location is presented as an
   exact quoted literal with a do-not-modify instruction, so the model stops
   rewriting `"santhoshpuram"` into the un-geocodable `"santhoshpuram, Chennai"`.

Planning and the location rule are *guided preferences*; **harvest and
block-repeat are the machine guarantees** — planning just makes the backstops
fire less often.

## Honest scope

- **Not included (planned next):** the keyless web search returns empty
  snippets by construction (`_parse_ddg_results` never captures snippet text) —
  a separate web_access fix that needs a real DDG-markdown fixture. The loop
  redesign already neutralizes its impact for "near me" tasks.
- The synthesis call is bounded by tier-3 + low `max_tokens`, not a hard
  timeout.

## Verification

New tests replay the exact failure: gathered data → `partial` with the data
(not `failed`); all-failed history stays `failed` (anti-hallucination gate);
repeat blocked but a new call allowed; plan echoed to later turns; location
presented verbatim. Full suite green (3361 passed).
