# v0.9.17 — Tools actually execute; the dashboard stays responsive

The most important release since the agent runtime shipped. It fixes a silent
defect that had neutered every built-in tool since v0.9.13, plus five
dashboard responsiveness/liveness bugs reported from live use.

## The big one: all 41 built-in tools were silent no-ops

v0.9.13 hardened security by running agent tools out-of-process — but the
subprocess executed each tool file directly as a script, while every curated
vault tool is module-style (`TOOL_META` + `run()`, no `__main__` block). The
script defined its functions and exited: **exit 0, empty output, reported as
success**. Every web search, fetch, file write, and places lookup silently
did nothing — which is why runs looped ("no objective credit for 10
iterations") and escalated to the operator: the model was told its calls
succeeded but never received any data, and the empty results even mis-credited
objectives.

Fixed three layers deep:

- **A stdlib-only subprocess runner** now imports the tool module and calls
  `run(**params)`, printing the JSON result. Script-style tools keep the old
  contract untouched (argv is rewritten for their `--params` parsing).
- **Truth-in-results:** exit 0 with no output is now a *failure* ("tool
  produced no output"), never a default success — this also re-arms the
  stuck-loop governor's failure trigger, so a broken tool gets steered or
  escalated after ~3 calls instead of 10 silent iterations.
- **Honest escalations:** the stuck question's "Tools tried" now reports every
  tool attempted since the last objective credit (it previously said "(none)"
  while a tool had been called repeatedly, because only failing tools were
  tracked).

The smoke harness now pushes a real module-style vault tool through the
subprocess path on uvicorn's actual serving loop — the exact gate this
regression slipped past.

## Dashboard responsiveness & liveness

- **Approving no longer freezes the screen.** Approve executes the authorized
  action (LLM extraction, pip installs, dry-runs) — it ran synchronously
  inside the UI event loop, starving the websocket (the brief
  connection-lost overlay). Gate resolution now runs in a worker thread; the
  UI stays interactive and reports completion when done.
- **Live panes no longer die until a manual refresh.** Cleanup was registered
  on a global disconnect hook, so any client's transient drop killed every
  pane's event subscription process-wide. Cleanup now fires only when a page
  is truly deleted.
- **"Needs you" rail redesigned** — stacked cards (risk pill, full wrapping
  title, right-aligned action) instead of one cramped truncated line.
- **Status dropdown** is easier to spot (icon added) and any render failure is
  now logged loudly instead of silently hiding the button.
- **Parallel task submission:** the chat Compose button no longer locks for
  the duration of a run (it was the de-facto serializer); the Supervisor's
  parallel width is tunable via `SYSTEMU_MAX_CONCURRENT_SHADOWS` (default 3).
  Screen captures remain one-at-a-time by design.

## Verification

Every fix was reproduced, test-driven, and re-verified live against a copy of
a real operator vault — including executing the operator's own `fetch_json`
through the fixed subprocess path (it now returns real data where it returned
an empty "success" before). Full suite: 2,970 passed, 0 failures.
