# v0.9.66 — R-A12b: the committed-effects ledger (IMPL-7)

**A handoff now says what it already did, not only what it still needs.** When a run stops
for the operator — parked on a harness escalation, a resume-confirmation, an ambiguous
effectful failure, or after hitting the iteration limit — its card was precise about the one
thing it needs next, but *silent* about the external effects it had already committed this
run. A handoff that names what it needs while hiding what it already did is not honest
(spec §5.6 / IMPL-7). This release closes that.

## What ships

- **Every precise-handoff / terminal-BLOCKED / stuck / partial card now enumerates the
  external effects already committed this run**, e.g.:

  ```
  Already committed this run:
    • created issue #412
    • imported invoice C
  ```

- **Rendered deterministically from persisted evidence — never model prose.** The ledger
  reads only the `confirmed` `ExternalEvidence` entries (the fail-closed bit set solely by the
  S3/R-A7 deterministic matcher, never an LLM — the same signal S4 credits on) and emits only
  each entry's short `detail` string, sorted by objective. An unconfirmed/advisory effect is
  **not** listed. The render is a pure, defensive function — a missing or malformed evidence
  store can never raise or fabricate a line.
- **Wired into the eight handoff/BLOCKED/stuck/partial finalizers** (the stuck-loop handoff,
  the harness-escalation parks, the resume double-submit guard, the ambiguous-effect park, and
  the max-iterations partial). Clean verified-done completions and bare shadow-failures are
  deliberately left untouched — a successful run needs no "still-needed" handoff.

## Compatibility & scope

- **Additive, presentation-only** — it renders a line from data S4 already persists; no new
  durable state, no migration, no change to credit/verification. A run with no committed
  effects renders nothing.
- **Scope note (R-A12 split).** This is the live, self-contained honesty half of R-A12b's
  pull-wiring. The rest — per-kind requirement→surface routing, the bundled upfront scope card,
  and resume-wiring the elicit ask path — is gated on landing R-A10's deferred mid-loop
  per-objective capability (so `requires_external_verification` stops being a no-op) and lands
  together as **R-A12c**; those surfaces are dormant until then, so building them now would be
  dormant machinery.
