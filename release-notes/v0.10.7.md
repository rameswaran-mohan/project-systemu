# v0.10.7 — Run costs survive a restart

A small but real robustness fix for the cost visibility added in v0.10.1.

Per-run cost was read from an in-memory ledger, so after the daemon restarted, a *completed* run showed
no cost at all — the running total was gone, and the run's snapshot (which briefly held it) is deleted
when the run finishes. Now each run's cost is written to a **durable** per-run file at completion, and the
cost view falls back to that file when the in-memory ledger doesn't have it. So you can restart the
daemon, open a run from yesterday, and still see what it cost. An in-flight run is unaffected — the live
ledger is always preferred when it's present, so nothing goes stale.

## Scope

- Per-run cost (Work rows, run detail) is now durable across a restart. The Home *daily* total is still
  computed from the in-memory ledger and remains session-scoped after a restart — a durable daily rollup
  is a separate, later item. Full suite green.
