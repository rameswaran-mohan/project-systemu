# v0.10.17 — An edit-safe test tier (GATE-TIER / DEC-14)

A developer-workflow improvement: there's now a fast test subset you can run **while code is still
being edited**, without spurious failures.

## The problem

Around 90 test files assert on a function or class body read via `inspect.getsource`. Those tests
compare against a source *snapshot* — so if a file changes while the suite is running (e.g. a
background agent editing it), `getsource` returns the new text and the assertion fails, even though
nothing is actually broken. The long-standing rule was "never run the full gate while files are being
edited."

## The fix

Rather than rewrite ~90 files, the test collector now **auto-tags** every getsource-reading test
`source_sensitive` (detected by module content — no manual marking). So:

```
pytest -m "not source_sensitive"
```

is a fast, **edit-safe gate** — 4,675 of the ~5,735 tests, with the ~1,060 source-snapshot tests
excluded — that you can run concurrently with source edits and trust. The full suite is unchanged;
this only adds a marker and a way to select against it.

Verified: the auto-tagger's detection contract is unit-tested, the marker is registered, and the
subset collects and excludes exactly the getsource tests. Full suite green.
