# v0.9.11 — UI Revamp Phase 5: Spine Consolidations (6-spine command center)

**Release date:** 2026-06-10
**Headline:** The dashboard's information architecture collapses from a flat 14-item nav into a **6-spine command center** — Home · Work · Shadows · Build · Insights · Settings — and the four content spines consolidate their duplicated surfaces. The biggest user-facing reshape of the revamp: one searchable Work spine, edit-in-place authoring under Build, a single memory source under Shadows, and decisions living in exactly one place (the Inbox).

> Phase 5 of the six-phase UI/UX revamp; builds on v0.9.10 (harness grant-resume) and the unified Decisions Inbox (v0.9.9). UX + internal-architecture release — agent behavior is unchanged. Regression-clean (**0 new test failures** vs the pre-existing baseline; **0 new** UI-style-lint violations) and put through a full live end-to-end verification.

## What landed

### IA shell → 6 spines
The nav is now the flat 6-spine rail (Home · Work · Shadows · Build · Insights · Settings) with exact, segment-aware active-route highlighting. **Inbox is demoted from the left nav** to where it belongs — the persistent right rail ("Needs you") + the full `/inbox` page + an always-visible **"Needs you (N)" header badge** (so pending gates stay reachable even when the rail collapses on narrow viewports). Every folded route stays registered — no dead links.

### Work — the workflow-centric spine
Scrolls + Activities + `/workflow/{id}` collapse into one **searchable list keyed on the workflow** (capture → scroll → activities → execution → done), with a status filter, name search, and clickable pipeline-stage chips that deep-link into the run. **Inspect-before-approve** (P4): scroll approval now flows through the unified Inbox card (risk · inline inspect · explicit "what Approve does" · highlighted safe-default) — the old blind one-click approve is retired. The **real side-by-side remediation card** replaces the dead flat panel, and the self-referential `fix_url` navigation loop is gone.

### Build — edit-in-place authoring
**One renderer per entity** (tools, skills) and **edit-in-place** — the Workshop page dissolves (its Tools/Skills/Shadows edit dialogs fold into the registries). Dependencies show inline in the tool row; `/skills` gains **deprecate / export / reactivate** (P9); the tool-forge gate routes to the rich Inbox review dialog (no double-forge); and the daily evolution-check **schedule cadence** is now visible/configurable in Settings.

### Shadows — one source (P10/P11)
Renamed from Army. **One** memory counter, **one** needs-consolidation rule (the long-standing 5-vs-10 disagreement is fixed — every surface now reads the engine's `BUFFER_THRESHOLD`), and **one** consolidation action that runs **off the UI thread** (P11 — the multi-second LLM call no longer freezes the dashboard). Shadow edit-in-place respects the active-shadow lock.

### Insights — one decision surface, one metrics home
The Insights "Actions" tab is retired — **decisions now render in exactly one place**, the Inbox (rail + page + Console mini-pane). Insights slims to the analytics views (memory health · flywheel · manual logs), and memory consolidation is surfaced from the Shadows spine.
