# v0.9.10 — Harness Grant-Resume (operator-approved capability escalations now suspend & resume)

**Release date:** 2026-06-10
**Headline:** A blocking reverse-harness escalation now **pauses the run and waits for you**, instead of pushing ahead with a fallback. When the Governor escalates a capability request (a new tool, a compute-ceiling bump, a skill, …) to the operator, the shadow run snapshots itself and parks; the request appears in the **Decisions Inbox**; and once you **Approve**, a daemon reconciler materialises the capability and **resumes the exact run where it left off** — with the granted capability live. **Deny** resumes with the agent's fallback. This completes the suspend→resume rails (the P2.3 leftover) that prior releases only half-built.

> Builds on **v0.9.9** (the UI/UX revamp: design system + shared command layer + unified Decisions Inbox + command-center shell). A PyPI install of 0.9.10 contains the full v0.9.9 feature set plus this release.

## What landed

### Blocking ESCALATE → snapshot + park (`systemu/runtime/shadow_runtime.py`, `supervisor.py`)
On a **blocking** harness ESCALATE the loop writes an `ExecutionSnapshot` (iteration, completed objectives, history slice, sticky notes) plus a `__HARNESS_PENDING__` marker, then returns a new `suspended_harness_escalation` status. `Supervisor._handle_result` treats that as a **park** — the activity is left assigned, **not** retried or dead-lettered — so nothing is lost while it waits. Non-blocking escalations keep the prior proceed-with-fallback behavior.

### Approve in the Inbox → materialise once → resume (`systemu/scheduler/jobs.py`, `daemon.py`)
A new in-daemon reconciler (`reconcile_resolved_harness_grants`, on the 15s tick) picks up a resolved harness gate, calls the Governor's provisioner **exactly once** (so a forged tool is built a single time), and calls `Supervisor.resume_after_grant(...)`. The resumed run peels a `__HARNESS_GRANT__` note and replays it through the **same** materialise-apply path an autonomous grant uses (`_apply_materialised_grant`, extracted as a shared helper) — so an operator-approved grant is byte-identical to an auto-grant, and the run **never re-arbitrates** (the operator's decision is authoritative). Idempotency is double-guarded (a distinct `harness_grant_dispatched` flag + the snapshot's grant note).

### Resume coords on the gate (`systemu/interface/harness_review.py`)
The harness gate now carries `activity_id`/`shadow_id` alongside `execution_id`/`kind`/`spec`, so the daemon reconciler can resume the precise parked run.

## Capability scope (honest)
- **Provisioned for real on resume:** **INPUT** (operator answer injected), **COMPUTE** (iteration/think budget bumped), **TOOL** (forged + deployed → callable in the same run), **SKILL** (authored to disk).
- **Replayed observation-only:** **ACCESS** and **SUBAGENT** — at **exact parity** with the *autonomous* GRANT path, which today only narrates these (no real sandbox patch / subagent spawn). Resume never claims more than the autonomous path delivers; making these provision for real is a future cut.

## Behavioral change to note
A blocking ESCALATE (the model's `blocking` field, default true) now **parks the run until you resolve the Inbox gate** — previously every escalation proceeded with the fallback. Un-acted gates leave runs parked (surfaced in the Decisions Inbox; there is no auto-timeout). If you run highly autonomously, confirm the agent sets `blocking` appropriately for your workflow.

## Known limitations / follow-ups
- **ACCESS/SUBAGENT** remain observation-only (autonomous-parity, above).
- **Forge-failure-on-Approve** currently narrates "Capability provisioned. Proceed." rather than a failure message when `materialise` returns un-materialised (cosmetic; the run is safe). Mapping it to a deny-style payload is a small follow-up.
- **No auto-timeout** on a parked, un-approved escalation (it stays visible in the Inbox).

## Migration
No vault migration. Additive and backward-compatible. All v0.9.0–v0.9.9 features untouched; non-blocking escalations behave exactly as before.
