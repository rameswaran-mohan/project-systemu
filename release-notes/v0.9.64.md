# v0.9.64 — CONC-MAP v1: durable-store writer-ownership map (DEC-10)

**A test-enforced map of who is allowed to write which durable store.** As the daemon grows
more background writers (reconcilers, the messaging thread, dashboard handlers, and soon R-A12's
external-wait reconciler), the risk is a second writer quietly appearing on a store that was
single-writer — a lost-update or a corrupt index with no compile-time signal. CONC-MAP v1 makes
that impossible to do silently.

## What ships

- **`docs/CONC-MAP.md`** — the writer-ownership map: the process/thread topology (one daemon
  process, many threads; CLI / `army execute` / Huey-worker as separate processes), and a table
  of every durable side-store → its writer(s) and home thread → its serialization (in-process
  lock + atomic replace, atomic-only, or none) → its risk. Plus a ranked multi-writer hot-spots
  section and the explicit R-A12 precondition.
- **`tests/test_conc_map_writer_ownership.py`** — the teeth. A source-scan pins the
  single-writer / known-writer sets (ExecutionSnapshot's three writers, OnTheTable's sole
  reconciler, the fatigue-metrics resolution writers, the R-P1 resolve-audit writer) and the
  atomic-write invariant on the key side-stores. **Adding a concurrent writer to a pinned store
  fails CI** — the failure *is* the DEC-10 concurrency-review checkpoint.

## Why now (the R-A12 gate)

R-A12 will add an `external_wait_reconciler` — a new background writer on `ExecutionSnapshot`,
the highest-risk store, running in a different process under the Huey queue where the current
process-local lock gives no protection. The moment that reconciler calls `write_snapshot`, the
CONC-MAP test fails until it is added to the allowlist and the map — forcing the review that
confirms it preserves the per-execution_id "only write while parked" invariant and closes the
cross-process gap for the new field. That is exactly the guardrail SEQ-2 / the amended DEC-10
called for, authored now so it adds zero spine latency.

## Scope

- **Documentation + a CI gate only** — no production code changes; every existing behavior is
  untouched. Full CONC-MAP enforcement (adding locks to the multi-writer hot-spots the map now
  names — the `decisions` store, vault collection indexes, the fatigue metrics, the login
  lockout) remains a precondition of R-P2 watchers / the R-W4 gardener, unchanged.
