# v0.9.18 — Answers in seconds: the quick lane

The first release of the "brilliant PA" roadmap. Until now, every chat ask —
even "what's the capital of France?" — went through the full factory pipeline
(scroll refinement → approval → activity extraction → shadow decision →
persona generation → execution): minutes of meta-work and four LLM calls
before the first useful action. That pipeline is the right machine for
*repeatable workflows*; it was the wrong default for one-shot asks.

## Quick answer — the new default lane

Chat's Compose now has a Mode control: **Quick answer (seconds)** — the new
default — alongside **Workflow run-now** and **Workflow queue** (the existing
pipeline, unchanged, one click away). A quick task runs a tight
reason-act loop directly against your enabled tools:

- **Same safety, same truth.** It can only call tools you've enabled (Gate 3),
  executes them through the exact same hardened path as the full runtime, and
  inherits the honest-results rules (a no-output tool call is a failure, not a
  phantom success). Hard caps end runaway loops honestly: iteration budget,
  wall clock, repeated-failure cutoff.
- **Answers arrive as answers.** The full markdown response renders in the
  chat thread (no more 120-character truncation), files it produced are
  listed underneath, and each iteration streams into the live panes with the
  usual expand-for-details arrows.
- **Promote what's worth keeping.** A successful quick answer offers
  **"Save as workflow"** — the prompt is refined into a scroll in Work
  (without re-running anything), where the normal approval → extraction →
  shadow flow makes it repeatable. The factory is the moat; now it's opt-in
  instead of a toll booth.

## Pick your brain

The reasoning tier has been a flash-class model by default — a deliberate
budget choice that was invisible and quietly capped result quality. Settings →
LLM Routing Tiers now offers one-click **presets** (`quality` / `balanced` /
`budget`), an advisory when your reasoning tier is a flash/free-class model,
and nothing changes unless you change it: no preset means exactly the same
models as before, and explicit tier settings always win.

## Results you can find

`files_produced` is now real end-to-end: tool calls that verifiably write
files (checked on disk, including when the sandbox redirects a stray path
into your output folder) are tracked through the run and surfaced in the chat
entry, the live feed's Outcome details, and the Status dropdown ("N file(s)").
Also fixed: workflow outcome summaries were silently persisted as empty
strings since v0.9.16 (a result-key mismatch) — task outcomes now actually
show what happened.

## Verification

50 new tests, all runnable without an API key (the quick lane's loop is
tested with an injected scripted LLM against real tool execution). Live-smoked
against a copy of a real operator vault. Full suite: 3,020 passed, 0 failures.
