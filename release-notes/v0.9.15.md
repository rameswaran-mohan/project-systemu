# v0.9.15 — Robustness (Wave 3) + UI polish (Wave 4)

A feature release that hardens the dashboard's failure paths and finishes the
board-faithful UI work: list filters everywhere, deep-linkable Insights tabs,
locally-bundled fonts (no CDN), and an accessibility pass. The full test suite
is green (no quarantined baseline failures), and a new boot-the-real-dashboard
smoke harness guards the release.

## Robustness (Wave 3)

- **Silent storage downgrades are now loud.** When a configured backend
  (sqlite/postgres/parallel) can't initialise, `AppState` falls back to the file
  vault. That used to be a log line only — a prod box pointed at postgres could
  quietly run on a local file vault, splitting data with no visible signal. The
  fallback now stamps a marker and the health banner raises a DANGER issue:
  "records written now will NOT be in your <backend> store".

- **Silent data/decision-path swallows made observable.** A targeted
  `except: pass` burn-down: the credential store (a corrupt file no longer
  masquerades as "no credentials"; keyring-delete failures log), the scheduler
  (a deleted schedule that fails to leave the index is reported, not orphaned
  silently), the runtime's stuck-loop terminal telemetry/refinery dispatch, and
  dangling skill/tool references in shadow creation now all log instead of
  vanishing. Control flow is unchanged — the operations stay non-fatal, they
  just stop being invisible.

- **Quieter logs.** The benign post-navigation NiceGUI traceback ("parent slot
  of the element has been deleted") is filtered out at the logger.

## UI polish (Wave 4)

- **Search + filter on every list.** A shared filter now backs Scrolls,
  Activities, Shadows, and Skills — Scrolls/Activities gained a status filter,
  Shadows gained search + status (it had none), and Skills' category filter runs
  on the same helper.

- **Deep-linkable Insights tabs.** `/insights?tab=memory|flywheel|events` opens
  that tab, and clicking a tab now updates the URL (no reload) so the active tab
  is shareable and bookmarkable.

- **Fonts bundled locally — no CDN.** Inter and JetBrains Mono (SIL OFL 1.1) now
  ship with the package and are served from `/assets/fonts` instead of Google
  Fonts. The dashboard renders the intended typography offline, with no
  third-party request on page load.

- **Accessibility.** A visible keyboard focus ring (the flatten layer had
  stripped focus outlines), plus accessible names (`aria-label` / `aria-current`)
  on the icon-only navigation.

## Tooling

- **Boot-the-real-dashboard smoke harness.** A pre-release E2E test boots the
  dashboard as a subprocess on uvicorn's actual serving loop and exercises it
  over HTTP — the loop where the v0.9.14 stuck-task regression hid. Run it before
  any release that touches runtime: `pytest tests/test_smoke_dashboard_e2e.py -m smoke`.

## Verification

Every change was test-driven (one focused suite per slice) and live-smoked
against the real dashboard. The long-standing baseline test failures were fixed
at the root (stale assertions updated to the shipped contract; singleton
order-pollution fixed), so the suite is green-means-green with zero quarantined
failures.
