# v0.7.4 — Tool Reconciler + `sharing_on init`

Closes two architectural gaps surfaced by the v0.7.3 UAT (see
`docs/superpowers/specs/2026-05-26-architecture-audit-six-patterns-design.md`
for full context).

## What's new

- **`sharing_on init`** seeds your working-dir vault from the wheel's bundled
  starter catalog (40+ tools, 25+ skills). Run once after `pip install systemu`
  in your chosen working directory. Idempotent — running it twice is safe.
  Use `--force` to overwrite existing entries or `--no-seed` for the directory
  structure only.
- **Tool Lifecycle Reconciler** runs inside the daemon on a 30-second interval.
  It scans for FORGED tools (regardless of the `enabled` flag) and advances
  them to DEPLOYED via the dry-run pipeline. Tools that were rotting at FORGED
  in v0.7.3 will now self-promote without operator intervention.
- **`[Dry-Run]` button** on the `/tools` page and matching
  **`sharing_on tools dry-run <id>`** CLI for manual operator-triggered
  advancement when an immediate dry-run is wanted.
- **Empty-vault warning** at daemon startup — when the operator hasn't run
  `sharing_on init`, the daemon emits a high-visibility warning recommending
  the command.

## Fixed

- **Bug #22 (v0.7.3 UAT):** Forged tools never reached DEPLOYED status.
  Root cause: the dry-run sweep filtered for `enabled=True` and only ran at
  daemon startup recovery. Reconciler removes both constraints.
- **Bug #21 (pre-v0.7.3):** Fresh pip install vault was empty. Root cause:
  wheel shipped catalog in site-packages, but daemon reads CWD.
  `sharing_on init` bridges the two.

## Migration

Existing v0.7.3 installs should run `sharing_on init` once to populate their
vault from the bundled catalog. Running it on a vault that already has tools
is safe (idempotent — only adds entries not already present). The
`SYSTEMU_AUTO_FORGE_TOOLS=true` flag continues to work as before for users
who rely on it; nothing changes there in this release.

## Known not-yet-fixed

The validator → auto-forge bridge (v0.7.3 Bug #14 fix) still requires
`SYSTEMU_AUTO_FORGE_TOOLS=true`. Pattern 3 from the architecture audit
addresses this in v0.8.1.

CLI commands invoked via the dashboard `JobManager` with no TTY still
auto-pick `actions[0]` ("safe default") rather than queuing for operator
input. This is Pattern 1 in the architecture audit; fix lands in v0.8.0.
