# v0.8.0.3 — Dashboard record → stop → refine pipeline actually works

Hotfix for the silent failure mode discovered during manual UAT of v0.8.0.2:
clicking "🔴 Record Session" → recording → "⏹ Stop & Analyze" produced **no
Scroll, no error, no toast** for every pip-installed user.

## Root cause

`AppState._project_root` was computed as
`Path(systemu.__file__).parent.parent.absolute()`. On pip installs this
resolved to `Lib/site-packages/` — completely outside the user's working
directory. The dashboard's `_stop_capture` then looked for the captures
directory at the wrong absolute path, found nothing, and silently skipped
dispatching the `sharing_on scrolls refine` follow-up job. Operators saw
recordings save successfully but no Scroll ever appear.

Three install/deploy modes hit this:
- **PyPI pip install + local mode** — `__file__` in `site-packages/` → broken
- **Git-clone editable install + local mode** — `__file__` in repo dir → worked
  by accident (CWD-style coincidence)
- **Docker mode** — `__file__` inside container → broken (no `.env` to walk to)

## Fix

New `_resolve_project_root(vault, config)` helper in `dashboard_state.py` with
a four-tier lookup so every mode works without operator intervention:

| Tier | Source | Handles |
|---|---|---|
| 1 | `SYSTEMU_PROJECT_ROOT` env var | docker (set in compose), explicit override, AND auto-set by `sharing_on daemon start` (CWD capture) |
| 2 | Walk up from `vault.root` looking for `.env` | pip install + git clone (local) |
| 3 | `vault.root.parent` | test env, docker without env var |
| 4 | `config.vault_dir`'s parent | last-ditch (vault has no `.root`) |

Companion changes so every tier fires correctly:

- **`sharing_on daemon start`** now propagates the resolved project root into
  the daemon subprocess as `SYSTEMU_PROJECT_ROOT` (Tier 1 fires automatically
  for local installs)
- **`docker-compose.yml`** sets `SYSTEMU_PROJECT_ROOT=/data` on all 4 dashboard
  services (Tier 1 fires automatically for docker)
- **Daemon startup banner** now prints resolved project root, vault path,
  `OPENROUTER_API_KEY` status, storage backend, mode, port — visible in
  both daemon log and foreground stdout
- **`_stop_capture` user-visible toasts** — always renders one of "Refine
  job dispatched for X", "No captures directory at Y", or "Capture stopped
  but no session directory was written". The old silent-skip is gone.

## Also fixed

- **`.env` UTF-8 BOM tolerance**: Windows PowerShell's
  `Set-Content -Encoding utf8` writes a BOM, which python-dotenv mis-parsed
  (the first key became `﻿OPENROUTER_API_KEY`). The CWD `.env` and
  legacy install-dir `.env` are now loaded with `encoding="utf-8-sig"` which
  strips a BOM if present and is safe for files without one.

## Migration

`pip install --upgrade systemu`. No `.env` changes needed. If you have
old daemons running from before this version:

```powershell
sharing_on daemon stop --all
sharing_on daemon start
```

The startup banner will now show resolved paths so you can verify
configuration in 1 second.

## Regression test

`tests/test_v0_8_0_3_project_root_resolution.py` (8 tests) locks down
each tier and the original pip-install failure mode.
