# v0.9.70 — R-A13a Stage 1: mid-loop requirement binder + bundled scope card

The first stage of turning the external-verification net on — a prep release that lands the
machinery with the net still **off**, so behavior is safe and unchanged for effect-verification.

## What ships

- **The agent now asks for genuinely-missing tool parameters — including nested ones — upfront.**
  When the model has chosen a tool for an objective, a deterministic pass checks the tool's full
  parameter schema against what was supplied. A missing required value (even one nested inside an
  object, which the previous top-level-only check couldn't detect) is gathered in one operator
  card, and the answer is bound back into the right nested position when the run resumes.
- **The card rides the existing, proven operator-input rail** — the same snapshot/suspend/resume
  path the runtime already uses — so a parked ask survives a restart and resumes cleanly, with no
  new suspend mechanism.
- **The external-verification net stays disarmed.** A per-objective external-verification signal is
  now computable, but it is held behind an internal three-state control (off by default) so no
  effect is stamped and nothing parks for verification. Later stages produce the evidence, measure
  the impact in a shadow mode, and only then arm it.

## Compatibility & scope

- **No behavior change with the net off:** a fully-provided call runs exactly as before; the
  external-verification stamp is never written. The full suite (5200 tests) is green.
- Array-*element* gaps aren't yet gathered by the new card (a follow-up); they fall through to the
  existing top-level required-parameter prompt, so nothing regresses.
