# v0.10.1 — Cost visibility (R-P3a)

Every run now shows what it cost. A per-run **token + currency** figure lands on Work rows and outcome
detail, the Home view shows a **daily total**, and prices are **operator-editable** in Settings — across
**both** lanes (the full agentic loop and the quick lane). No spend caps (that comes later); this release
is about *seeing* the cost, honestly.

## What ships

- **Per-run cost, both lanes.** The token counts already captured on every model call are now accumulated
  against the owning run and priced. A Work row shows e.g. `$0.012 · 38k tok`; the outcome detail breaks
  it down by model; the Home view sums today's runs into a daily total that **reconciles exactly** with
  the per-run figures.
- **Operator-editable prices.** A Settings price editor lets you set per-model rates (and your own
  currency); a change takes effect **immediately**, no restart. Shipped defaults cover the models the
  router actually uses, quoted in USD — override to your currency as you like.
- **Money honesty — never a guessed number.** An **unknown** model shows its token count with the cost
  rendered **"unknown"**, never a fabricated figure. All money is computed in exact decimal, never
  floating point. Cost is shown separately from verified/claimed status — a cost figure never implies a
  run succeeded.

## Under the hood (why "both lanes" is the hard part)

- The accumulator hooks the model-call chokepoint itself (where the token counts are known), attributing
  each call to its run via the ambient run id. This closed a subtle trap: the sync model-call path crosses
  worker-thread boundaries that do **not** carry that ambient id by default — without care, *every* cost
  would have been orphaned (attributed to no run). The run id is now propagated across those boundaries,
  and the quick lane sets it too, so no cost is lost or misattributed. Regression tests pin it closed.
- The run's cost rides its record across a suspend→resume, so a parked-then-resumed run keeps its cost.

## Hardened before release (adversarial review found + fixed 5 real bugs)

A focused adversarial pass on the cost accounting found and fixed five correctness bugs at root:

- **The planner's model call was orphaning its cost** — it crossed the same worker-thread boundary as the
  main loop but hadn't carried the run id across it, so an entire planning call's cost vanished. Fixed.
- **A parked-then-resumed run could lose or fragment its cost** — a resume starts a fresh run id, and the
  durable cost was being overwritten with only the post-resume portion (and a stale entry could
  double-count in the daily total). A resumed run now re-attaches its earlier cost and keeps one coherent
  total — nothing lost, nothing double-counted.
- **Money-honesty edges:** a negative price override is now rejected (it could have produced a negative
  total); a currency written in a different case (e.g. "usd") now matches its shipped default instead of
  wrongly showing "unknown"; and a sub-agent's cost no longer double-seeds its parent's.
