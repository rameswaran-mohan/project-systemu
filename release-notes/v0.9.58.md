# v0.9.58 — S1b: the action gate goes live (per-tool + MCP first-use) + approval-fatigue metrics

**The deterministic action gate — built as a pure evaluator in S1a (`evaluate_action`) — is now
wired into the LIVE tool-execution path.** Effectful/forged tools prompt the operator
(Deny / Approve-once / Always-allow); MCP tools gate on first use even when they self-declare
read-only; the forge-time dry-run fails closed on anything it can't prove non-egress; and the
approval-fatigue baseline is captured from the day the gate goes live.

Milestone **S1b (R-A5b)** of the Open-World Go-Getter build (spec `UNIFIED-v2` §5.7 + IMPL-1…4,
plan §R-A5). All effectful actuation now passes through the gate before it runs.

## What ships

- **Per-tool live gate (IMPL-1 — the crux).** A new `ToolSandbox._maybe_gate_tool`, invoked at the
  single `execute_tool` chokepoint (so the v2 registry fast-path can't slip an effectful tool past it),
  builds an `ActionContext` from the tool's `effect_tags` and calls `evaluate_action`. A
  `REQUIRE_APPROVAL` verdict posts a `gate_type="tool"` card and parks the run; it resolves through the
  **same** command-gate park→resume rail (zero new resume machinery). Approvals key on a
  **`tool_signature`** = `(name, body-hash, sorted effect_tags[, host-class])`, so a re-forge or tag
  change re-gates. **Always-allow** now creates a genuine standing approval (`store.approve`);
  Approve-once is single-use.
- **MCP first-use gating.** A discovered/registry/first-use MCP tool is gated regardless of a
  self-declared `readOnlyHint` (`classification_trusted` derived from whether the tool hash was ever
  pinned). Already-trusted/pinned tools are not re-gated — no fatigue.
- **Fail-closed forge dry-run (IMPL-3).** The forge-time dry-run used to execute freshly-forged,
  unreviewed code with full network privileges, guarded only by a name heuristic. It now **skips** any
  tool it cannot affirmatively prove non-egress — net/shell/`unknown`/undeterminable tags all skip
  (`effect_tags`, or `classify_source` of the body when unstamped); only a tool whose tags are a
  non-empty subset of `{local_read, local_write, local_delete}` proceeds. This closes a real
  live-egress hole before S2's OS jail exists. (`net_read` is deliberately in the skip set — a dry-run
  must not phone home even for a read.) A companion reaper fix ensures a `skipped` re-validation can
  never un-condemn an already-failed tool (which previously hung the blocked activity in `partial`).
- **Approval-fatigue metrics (PLAN-11 / DEC-11).** A new append-only `runtime/metrics_store.py`
  (atomic write, defensive read) plus **best-effort** counter hooks at the decision post/resolve
  chokepoints: gate cards created/resolved, resolution-latency histogram, always-allow grants, denies,
  bulk-approve detection (≥2 resolutions <2s apart), asks created/resolved. Every write is guarded so a
  metrics failure can never break the decision flow. No UI this release — the baseline exists from day
  one; R-P3b / R-A13.5 surface it later.
- **`net_read` stays frictionless.** A plain network-reading tool is `ALLOW`, not over-gated (fixed a
  spec-level bug where a net-tag was mis-mapped into a mutation escalator).

## Test infrastructure (shipped with the live gate)

- **`tests/conftest.py`** bounds the quick-lane gate block-poll (`_poll_command_choice` /
  `_ask_operator_inline`) to ~2s in tests, so a fired-but-unresolved gate **fails fast** (Deny) instead
  of stalling 5 minutes — the test-side of a live gate. Pre-resolved decisions are unaffected.
- Test tool helpers now tag their tools via `classify_source`, mirroring the production forge backfill,
  so low-risk local tools correctly resolve to `ALLOW`.

## Tests

- Full suite: **4350 passed, 22 skipped, 0 timeouts** (~5:25). The block-poll bound holds without any
  `pytest-timeout` dependency (that tool was diagnostic-only during development). The lone full-run
  failure is a **pre-existing, non-S1b** test-isolation bug (a different test forges into the real seed
  vault and trips `test_index_matches_disk`; passes on a clean vault — flagged as a separate follow-up).
- Every S1b piece landed test-first (RED→GREEN) and passed independent spec + code-quality review; the
  review chain caught and fixed at source a `net_read` over-gate, a dry-run fail-open hole, and a reaper
  regression.

## Operator tryout (definition-of-done, session #2)

The end-to-end **park → resume** across the quick-lane block-poll is operator-verifiable only (not
gate-testable). Answer a real forged-tool gate card from the dashboard — **Deny**, **Approve-once**, and
**Always-allow** — and confirm the run parks and then resumes.
