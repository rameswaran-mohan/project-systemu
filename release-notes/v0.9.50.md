# v0.9.50 — chat & navigation quality-of-life

A batch of dashboard ergonomics so running and steering tasks feels direct.

## Chat page

- **Enter runs the task** (Shift+Enter inserts a newline) — no more reaching for
  the Run button or Ctrl+Enter.
- **"Workflow — run now" is the default mode** (the common case), with Quick
  answer and Queue still one click away.
- **The running task is surfaced immediately** — after you submit, the new card
  appears and scrolls into view, so you see the job take off without scrolling.
- **Per-job switches on each card** — a **kill** icon on a running job and a
  **restart** icon on a finished / killed / stuck job, top-right of the card.

## Navigation

- **Approving a forged tool returns you to the Inbox** you started from, instead
  of stranding you on the Tools page (and it no longer leaves a stale
  client-deleted warning behind).
- **A "← Back to Chat" button on the Inbox** for a one-click hop back to the
  composer.

## Under the hood

Pure interface ergonomics — no runtime/behavior changes. The kill switch reuses
the existing cooperative-cancel token; the restart re-runs the task's prompt
through the normal submission path. No eval/paper changes.
