# v0.9.71 — R-A13b Stage 2 (part 1): the shadow park-surface meter

The second stage of turning the external-verification net on. This release adds the **measurement
instrument** — a record-only meter that, in an internal shadow mode, measures how the net *would*
behave without changing any behavior. The net stays **off**; nothing is stamped, credited, asked,
or parked.

## What ships

- **The system can now measure its own external-verification park surface — safely.** When the
  internal control is in shadow mode, for each objective that *would* require external verification,
  the runtime now produces the verification evidence and records whether it *would* have credited the
  effect or *would* have needed to park it for operator confirmation — broken down by effect type.
  This is the reading that will decide, in a later stage, when it is safe to actually arm the net.
- **Record-only, by construction.** The meter never credits an objective, never raises an operator
  card, and never blocks a run. It only records. A run's outcome is exactly what it was before —
  the meter observes the run, it does not steer it.
- **The measurement is faithful, not optimistic.** Each shadow measurement is evaluated exactly as
  the *armed* net would evaluate it, so the recorded "would-credit / would-park" numbers are a true
  predictor of the real behavior — not a rosier one. (Two review-surfaced ways the shadow reading
  could have diverged from the armed reading were closed at the root before shipping.)
- **Instrumentation stays out of the operator's way.** Shadow measurements are kept to the internal
  metrics report only; they never appear in a run's "already committed this run" summary.

## Compatibility & scope

- **No behavior change with the net off (the default):** runs with the net off — and runs with the
  net fully armed — are byte-identical to before. Only the internal shadow mode records anything, and
  what it records changes no run outcome. The full suite (5219 tests) is green.
- The tools that participate in external effects don't yet emit the verification envelopes the meter
  reads, so in practice the shadow reading is conservative today; populating those envelopes and
  producing the shadow report across real runs is the next step.
