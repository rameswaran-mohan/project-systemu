# v0.9.59 — G1: objective-graph persistence substrate + SnapshotMigrator (DEC-9)

**An execution's objective graph is now durable across suspend/resume.** The snapshot carries
the objective graph + its id-allocator floor, the resume path rehydrates it (falling back
byte-identically to the static scroll tree when absent), and a versioned `SnapshotMigrator`
migrates older snapshots forward and honestly refuses a newer-than-supported one — never a
silent fresh start.

Milestone **G1 (R-A2)** of the Open-World Go-Getter build (spec §5.2 + §15.2 DEC-9). This is the
*persistence substrate* the graph-mutating milestones (the open-world planner + backward-chained
requirements + discovery-inserts) build on: it ships the read side + the migrator so any future
graph mutation survives restart. No code in G1 yet writes a non-empty graph — the substrate is
proven end-to-end with synthetic snapshots.

## What ships

- **`Objective` carries provenance (SPEC §5.2).** Two additive, default-safe fields:
  `origin: Literal["planner","discovery","retry","backchain"] = "planner"` (where a node came
  from) and `requires_external_verification: bool = False` (whether its effect demands an external
  ground-truth check). Legacy on-disk objectives validate unchanged. (`requirements` is
  deliberately deferred to §5.3 / the requirement-binder milestone — the substrate round-trips the
  whole `Objective`, so it survives restart for free when it lands.)
- **`ExecutionSnapshot` persists the graph.** New `objective_graph: list[Objective]`,
  `next_objective_id: int`, and `schema_version: int`, wired through the hand-written
  serialize (`_to_dict`) + load (`read_snapshot`) + `capture_from_context` factory. `write_snapshot`
  stamps the current schema on every write, so a freshly-written snapshot never triggers a needless
  migrate-on-read.
- **SnapshotMigrator (DEC-9) — `runtime/snapshot_migrations.py`.** A stdlib-only leaf module with a
  `{from_version: migrate_fn}` registry chained at load. `== CURRENT` → fast path;
  `< CURRENT` → back up the file once (`.bak`) then migrate ascending; `> CURRENT` (or an
  unintelligible / sub-1 `schema_version`) → **raise `SnapshotRefused`** — an honest refusal that
  the read path re-raises past its blanket `except` rather than degrading to a silent "no snapshot →
  start fresh". The migrator never rewrites the original file.
- **Refused snapshots never silently start fresh.** All three `read_snapshot` callers handle the
  refusal: the shadow-runtime resume block (the single fresh-vs-resume chokepoint) returns a
  **`structural_failure`** result, which the supervisor routes straight to a terminal state instead
  of the generic failure→retry path (which would have re-submitted the activity *fresh* and
  re-executed the parked run's effects). The decision-resume and grant-resume callers log loudly and
  rely on the chokepoint. This closes the DEC-9 hazard end to end.
- **Resume rehydrates the mutated graph — AC6-safe.** A new `_resolve_objectives_for_run` folds the
  persisted-graph branch and the v0.9.35 param-substitution seam-fix into one precedence-ordered
  assignment (persisted graph > param-substitution > static scroll tree). When no graph is persisted
  (every run today), it returns the scroll's objectives **by identity** — so **AC6** (a never-mutated
  run produces a byte-identical schedule) holds by construction. The completion denominator and the
  id-allocator floor are re-derived after the fold.

## AC6 — the byte-identical guarantee

AC6 (§5.2) is the milestone's hard safety net: a never-mutated run must schedule identically whether
run fresh or resumed. It holds because the per-iteration schedule is a pure function of the objective
list, and the no-graph path preserves that list by object identity. Verified by an executed
fresh-vs-resume schedule comparison across a single-objective scroll, a multi-objective scroll with
`depends_on` edges, and action-block mode — byte-identical in every case.

## Deferred to the graph-mutation milestone (unreachable today)

Nothing in G1 writes a non-empty `objective_graph`, so these are latent until the planner/discovery
milestone wires runtime graph mutation: merging a param-substitution *onto* a persisted graph
(rather than dropping it); the stuck-answer-block completion denominator; and an honest refusal for a
malformed persisted-objective dict. All recorded for that milestone.
