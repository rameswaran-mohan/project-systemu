# v0.10.3 — Quick-lane runs show their cost too

A small completeness follow-up to the cost visibility in v0.10.1. The Home "today's
spend" total spans **both** lanes, but only workflow runs had a per-run cost figure
(on their Work rows). Quick-lane runs — the ones you fire from the chat box — showed
up in the daily total with no way to see each one's individual cost.

Now every quick-lane chat entry carries a cost chip (`$0.012 · 38k tok`), matching
the Work-row chip. So the daily total's constituents are all visible per-run, across
both lanes — no lane is a cost blind spot. As with everywhere else, an unknown model
shows tokens with the cost as `—`, never a made-up number, and cost is shown
separately from success.

The run's cost is stored on its chat entry, so the chip survives a page reload.

Full suite green.
