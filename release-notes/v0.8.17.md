# v0.8.17 — free out-of-box web search (ddgs + cache/backoff + Tavily/Exa + fail-fast)

Web search now works on a fresh install with **zero config** — no API keys required.
Previously, every web-dependent chat task silently failed and ground to MAX_ITERATIONS (30 steps)
with nothing in the event log to explain why.

## Root cause

The old free-tier provider (`DuckDuckGoLiteProvider`) was a DIY HTML scrape of
`https://lite.duckduckgo.com/lite/`. DuckDuckGo blocks automated scraping, so it
returned empty results on every call. The `web_search` tool reported `success: False` with no
explanation, the agent retried through the full 30-iteration budget, and the task failed — slowly,
expensively, and with a cryptic timeout as the only signal to the operator.

Every chat task that needed current or factual web data failed this way on any deployment without
API keys (the common case for a fresh install).

## The fix

**Free floor — bundled `ddgs` keyless metasearch.**
`ddgs` (validated v9.14.4) is now a hard dependency. It is an actively maintained keyless
metasearch library whose `backend="auto"` mode rotates across Bing, Google, Mojeek, Startpage,
Yandex, and others, making it far more robust than a single-target scrape. `DdgsProvider` wraps it
with:
- One backoff/retry on `RatelimitException` / `TimeoutException` before giving up.
- Graceful degradation: no unhandled exception is ever raised from the provider; worst case is an
  empty list.
- `available()` always returns `True` — it is the unconditional free floor of the chain.

**POC validation.** `ddgs` was validated on a live network before shipping: query
`"best hair salons near Mumbai"` → 5 results in under 2 s, including titles, URLs, and snippets.
The call also works correctly inside the async executor (no event-loop conflicts).

**Optional keyed providers for guaranteed reliability.**
Set either or both of these environment variables for an API-backed provider that sits above `ddgs`
in the chain:

| Env var | Provider | Free tier |
|---|---|---|
| `SYSTEMU_TAVILY_API_KEY` | Tavily | ~1 000 queries / mo |
| `SYSTEMU_EXA_API_KEY` | Exa | ~1 000 queries / mo |

Both have no-credit-card free tiers. Either or both give you a guaranteed-reliable search backend.

**Full provider chain (first non-empty wins):**
`Tavily → Exa → Brave → Serper → ddgs`

Keyed providers are skipped if no key is set. `ddgs` is always the last resort and is always
available.

**Short-TTL result cache + rate-limit protection.**
`search()` now caches results in memory for 5 minutes (300 s). Repeated identical queries within
the window are served from cache. Empty / degraded results are not cached, so transient rate-limit
failures do not poison subsequent calls. The cache resets at 128 entries.

**Model-facing `note` on degraded/empty results.**
When `web_search` returns empty results, the tool output now includes a `note` field that tells
the model: web data is unavailable, do NOT retry the same query, and if web data is essential,
fail the task with that reason. This prevents the agent from spinning on a broken search step.

**Fail-fast after 3 consecutive degraded searches.**
The ReAct loop now counts consecutive degraded search-tool results (the whole provider chain
failed/empty — not just a fuzzy-match zero-result). After 3 in a row, the run terminates
immediately with a clear failure reason:

> Web search capability unavailable — search backends failed 3x. Set SYSTEMU_TAVILY_API_KEY or
> SYSTEMU_EXA_API_KEY for reliable search.

This replaces the previous behaviour of looping silently to MAX_ITERATIONS (30 steps, ~2–3 min)
before the operator noticed anything was wrong.

## Migration

```
pip install --upgrade systemu==0.8.17
```

`ddgs` and its prebuilt transitive dependencies install automatically — no compiler is required.
Network access is needed at install time.

No vault reseed required. Existing vault contents, activities, and executions are fully compatible.

**Optional (strongly recommended for production):** set one or both free-tier API keys in your
`.env` or shell profile to get a more reliable search backend above `ddgs`:

```
SYSTEMU_TAVILY_API_KEY=tvly-...   # https://tavily.com — free tier ~1000 queries/mo
SYSTEMU_EXA_API_KEY=...           # https://exa.ai    — free tier ~1000 queries/mo
```

Restart the daemon after setting keys.
