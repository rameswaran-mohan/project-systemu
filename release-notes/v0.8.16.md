# v0.8.16 — live, expandable, origin-partitioned event panes

The Console's event feeds are now **live** (no page refresh), **split by trigger
origin**, and **expandable** to show each step's reasoning, tool I/O, and the raw
LLM response. Closes the "not live / wrong pane / no LLM responses" issues.

## What was wrong

- **Not live.** The Supervisor pane called `_pane.refresh()` from inside the
  EventBus subscriber callback — which runs on the *publishing* (background)
  thread. NiceGUI never pushed those DOM updates, so the operator had to refresh.
  (The EventBus docstring says the UI must use **deque + timer**.)
- **Wrong pane / divergent sources.** The Supervisor pane read the EventBus; the
  "Event Log" pane read `event_log.jsonl`. Execute/runtime events reached the bus
  but not the file, so they showed in one pane and not the other.
- **No LLM/tool detail.** LLM interactions were only `logger.debug`'d; events
  rendered only a one-line message with no way to inspect reasoning or tool I/O.

## What changed

**1. Canonical `origin` taxonomy, propagated end-to-end.** `origin ∈ {chat,
capture, manual, scheduled, system}` is stamped at the *trigger* entry point
(chat → `chat`; capture flow → `capture`; operator execute → `manual`; scheduler
→ `scheduled`; recovery/retry/restore → `system`) and carried through
`Supervisor.submit(origin=…)` → the worker → `ShadowRuntime.execute(origin=…)` →
**every** published event (via `_stamp`). `Activity.origin` is the durable
carrier; `coerce_origin()` maps legacy submit reasons.

**2. Both panes are live and partitioned.** The pane now follows the EventBus's
contract: the subscriber callback only appends to a bounded deque, and a
**UI-thread `ui.timer(0.5s)`** is the sole refresh driver — so updates appear
instantly without a page refresh. Panes filter by origin:
- **Supervisor pane** → `chat` only.
- **"Event Log" → renamed "Manual Logs"** → `capture` / `manual` / `scheduled`.
- **System** events (recovery/retry/restore) are hidden by default behind a muted
  **"Show system"** toggle per pane. (The `/insights?tab=events` route is
  unchanged — only labels were renamed.)

**3. Expandable events with reasoning + tool I/O + raw LLM.** The runtime now
publishes a bounded per-iteration event carrying a `details` dict (reasoning,
action, tool name, tool params, tool result). Each such event renders inside an
expansion **arrow**; expanding shows the detail, plus a **"Show LLM response"**
button that **lazily** loads the raw completion from a per-execution transcript
(`vault/executions/<exec_id>/llm_transcript.jsonl`) — so the live feed/ring buffer
stays small. Plain notification events render as flat rows (no arrow).

## Migration

`pip install --upgrade systemu==0.8.16` and restart the daemon. Existing activities
gain an `origin` field defaulting to `"manual"`. Chat runs now appear live in the
Supervisor pane; captures / manual / scheduled runs in Manual Logs; expand any
runtime event to inspect reasoning, tool I/O, and (on demand) the raw LLM response.
