# v0.9.57 — GrantedRoots grant + confinement store (G2)

**A net-new filesystem confinement primitive: the operator grants directories, and
every resolved path is checked WITHIN a granted root after NTFS-safe
canonicalization. Purely additive — nothing consumes it yet; it's the substrate the
Situational Inventory, the Reference Resolver, and the `access` requirement type will
rest on.**

Foundation milestone **G2** of the Open-World Go-Getter build (spec `UNIFIED-v2` §5.4
/ §13, review finding HIGH-3 — confirmed net-new: no grant/confinement primitive
existed).

## What ships

`runtime/granted_roots.py`:

- **`GrantedRootsStore(base_dir)`** — a persistent set of operator-granted root
  directories with `grant` / `revoke` / `list_roots` / `is_granted_root` /
  **`is_within_granted(path)`**. Side-store at `<vault>/granted_roots.json` (atomic
  write, defensive read — a broken/absent file yields no grants, never an exception).
- **`canonicalize(path)`** — the confinement-safe canonical form (IMPL-9): the FINAL
  path via `os.path.realpath` (follows `..` / symlinks / junctions / reparse chains
  to the end), 8.3 short-name aliases (`PROGRA~1`) expanded on the existing prefix via
  `GetLongPathNameW` (ctypes, Windows), then case-folded / normalized
  (`os.path.normcase`). A raw-string prefix check is **never** the boundary.
- The boundary test uses **`os.path.commonpath`**, which respects the component
  boundary — so `/granted-evil` is correctly **not** within `/granted`, and a
  different-drive path is out.

## Why it matters

This closes the review's **HIGH-3** gap (the spec's `access`/confinement claims
rested on a store that didn't exist) and folds in **IMPL-9** (the NTFS
canonicalization the v2.1 review flagged: `realpath` alone is not the boundary on
Windows — junctions, 8.3 aliases, and case-fold each need handling).
