# v0.9.25 — Setup asks which provider (not just OpenRouter)

A fast follow to v0.9.24. The per-tier provider machinery shipped, but the
**interactive** `sharing_on setup` still only asked for an OpenRouter key —
so picking Google/OpenAI/Anthropic/Ollama, or mixing providers per tier,
was only reachable via CLI flags. Now the wizard asks up front:

```
Step 1 — Which LLM provider?
  1. OpenRouter  (recommended — one key, 200+ models)
  2. Google      3. OpenAI      4. Anthropic      5. Ollama (local, no key)
  6. Different provider per tier (advanced)
```

- **Choices 1–5** use that provider for all three tiers — **one credential
  covers all three** (the "same token for every tier" case). OpenRouter
  keeps the simple one-key + preset flow unchanged.
- **Choice 6** walks the three tiers, letting you pick a different provider
  each, with "same as tier 1" shortcuts and **credential reuse** when a
  provider repeats (you're never asked for the same key twice).
- Ollama collects a base URL instead of a key; Anthropic notes if the
  optional package isn't installed.

No data-model change — this just surfaces what v0.9.24 already wired.

## Verification
5 new interactive-flow tests + the existing setup suite, all keyless. Full
suite green.
