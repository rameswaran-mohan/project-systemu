# v0.8.10 — Tiered web capability + fresh posture

v0.8.10 replaces the ad-hoc web/browser tools with a single **tiered web
capability** that works out-of-the-box on a bare `pip install`, plus a
re-postured fresh seed (curated tool + skill packs, one Wild Card shadow,
zero runtime state).

The capability is organized as three tiers that escalate only as needed, so
the cheapest path that can answer the request is the one that runs:

- **T0 — fetch + readability.** An `httpx` GET with realistic headers, size and
  content-type guards, then pure-Python readability extraction (stdlib
  `html.parser`, no BeautifulSoup) that strips script/style/nav/footer chrome
  and resolves relative links. Works the instant the daemon boots.
- **T1 — multi-provider search.** Keyed providers (Brave, Serper) are preferred
  when an API key is present; otherwise it falls back to a free DuckDuckGo-lite
  scrape. The result is tagged with the provider used and a `degraded` flag so
  callers know when they are on the free floor of the chain.
- **T2 — hardened headless browser.** Accessibility-tree-first interaction,
  **headless-only** (never attaches to the operator's Chrome), a context pool
  with a concurrency cap, and a domain allow/deny policy. Used for JS/SPA pages,
  screenshots, and interactive flows.
- **web_act** — an LLM-driven interaction loop on top of T2: snapshot the
  accessibility tree, let a Tier-2 model pick one action (CLICK / TYPE / READ /
  DONE), execute it via role+name locators, observe, repeat. Bounded by
  `max_steps`; refuses to type into password-named fields.

## Auto-provision

On first daemon start, if the Playwright chromium binary is missing, the daemon
spawns a **background** `playwright install chromium`. T0 fetch and T1 search
work immediately during the install, so the operator is never fully blocked —
full browsing (screenshots, JS pages, interaction) comes online once chromium
finishes (~30–60s). The probe is idempotent. Set
`SYSTEMU_SKIP_BROWSER_AUTOINSTALL=true` to opt out.

## Four stable web tools

The web surface is now exactly four tools:

- **web_search** — multi-provider search (T1).
- **web_read** — read a page; T0 fetch first, auto-escalates to the headless
  browser for JS/SPA pages.
- **web_screenshot** — full-page screenshot via headless chromium (T2).
- **web_act** — drive a page to accomplish an instruction (T2 + LLM loop).

**Breaking:** the old `web_extract_text`, `web_extract_table`,
`browser_navigate`, `mouse_click`, and `mouse_drag` tools are **removed**.

## Packaging fix

The wheel now ships the vault seed (`vault/**/*.json`, `vault/**/*.md`) and the
tool implementations (`vault/tools/implementations/*.py`) via
`include-package-data` + `package-data`. Previously a bare `pip install` could
land without the seed or tool code; a fresh install now boots with the full
curated capability set.

## Classifier fix

`classify_dry_run_error` now accepts a structured `missing_packages` list and
prefers it over regex sniffing of the error text. A dry-run that reports its
missing packages directly is classified as `DEP_PENDING` with the real package
name instead of falling through to `unknown`. The regex fallback is unchanged
for callers that pass only error text.

## Fresh posture

v0.8.10 re-postures the app as a clean first-run experience:

- **Curated 38-tool starter pack.** Dropped 15 dev-cruft tools (github PR/commit/
  workflow scrapers, NSE/finance fetchers + RSI/SMA calculators, Docker Hub
  metadata, reddit scraper, email search, and `send_email`).
- **Curated skill pack.** Dropped task-specific demo skills (clinical-trial
  search/extraction, ci-cd pipeline analysis, finance/email/weather automation).
- **Zero runtime state.** No scrolls, activities, or evolutions in the seed.
- **One Wild Card shadow**, wired to the curated tool ids.

## Backward compatibility / migration

This release re-postures the app **fresh**. The old web tool names are gone, so
existing installs should reset their vault rather than upgrade in place — any
saved scroll/skill referencing `web_extract_text`/`web_extract_table`/
`browser_navigate`/`mouse_click`/`mouse_drag` will no longer resolve. New
installs need no migration.

New environment variables (all optional):

- `SYSTEMU_BRAVE_API_KEY` — enable the Brave search provider.
- `SYSTEMU_SERPER_API_KEY` — enable the Serper search provider.
- `SYSTEMU_BROWSER_MAX_CONTEXTS` — headless browser concurrency cap (default 3).
- `SYSTEMU_WEB_ALLOW_DOMAINS` — comma-separated allow-list (restricts browsing).
- `SYSTEMU_WEB_DENY_DOMAINS` — comma-separated deny-list.
- `SYSTEMU_SKIP_BROWSER_AUTOINSTALL` — skip the background chromium install.

## Migration

`pip install --upgrade systemu`. T0/T1 work immediately; the headless browser
auto-installs in the background on first daemon start. Existing installs should
reset their vault to pick up the fresh posture and drop references to the removed
web tools.
